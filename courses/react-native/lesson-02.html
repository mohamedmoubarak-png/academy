.1 طريقتان لبدء مشروع RN Two Paths

عندك طريقتان لإنشاء مشروع React Native:

⚡ Expo ✅ الموصى به
  • ✅ إعداد سهل وسريع
  • ✅ لا تحتاج Android Studio
  • ✅ شغّل على جهازك فوراً
  • ✅ مكتبات جاهزة (كاميرا، GPS...)
  • ✅ للمبتدئين والمحترفين
🔧 React Native CLI
  • ✅ تحكم كامل
  • ❌ يحتاج Android Studio
  • ❌ إعداد معقد
  • ❌ للمشاريع المتقدمة جداً
  • ⚠️ ليس لهذا المسار
قرارنا: سنستخدم Expo في كل هذا المسار. هو الخيار الأذكى للتعلم وحتى للمشاريع الحقيقية!

.2 ما هو Expo؟ What is Expo?

Expo منصة مبنية فوق React Native تُسهّل التطوير بشكل كبير. تتكون من ثلاثة أجزاء:

الجزءما هو؟الاستخدام
Expo SDKمجموعة مكتبات جاهزةكاميرا، GPS، إشعارات...
Expo CLIأداة سطر أوامرإنشاء المشاريع وتشغيلها
Expo Goتطبيق على جهازكمعاينة التطبيق فوراً
📱 Expo Go = كاميرا حية:

بدل ما تبني وتنشر كل مرة، تشغّل Expo Go على جهازك وكل تغيير في الكود يظهر فوراً على الشاشة!

.3 الخطوة 1: تثبيت Node.js Install Node.js

React Native يعمل على Node.js. نحتاج الإصدار 18+:

1

حمّل Node.js LTS

افتح nodejs.org واختر الإصدار LTS (Long Term Support). هو الأستقر.

2

ثبّت وتحقق

بعد التثبيت، افتح Terminal أو CMD وشغّل:

node --version
v20.11.0
npm --version
10.2.4
لو ظهر الإصدار، Node.js جاهز ✅. لو ظهر خطأ، أعد التثبيت.

.7 هيكل ملفات المشروع File Structure

بعد إنشاء مشروع Expo، هذا ما ستجد:

MyFirstApp/ ├── app/ ← ملفات الشاشات (Expo Router) │ └── index.tsx ← الشاشة الرئيسية ├── assets/ ← صور وأيقونات │ ├── icon.png │ └── splash.png ├── components/ ← مكوناتك المخصصة ├── App.js ← نقطة البداية ├── app.json ← إعدادات التطبيق (الاسم، الأيقونة...) ├── package.json ← الحزم والسكريبتات └── babel.config.js ← إعدادات Babel
أهم ملف: App.js
📄 App.js — نقطة البداية
import { StatusBar } from 'expo-status-bar';
import { StyleSheet, Text, View } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text>Open up App.js to start working!</Text>
      <StatusBar style="auto" />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
لاحظ: styles.container يستخدم flex:1 — هذا يجعل الـ View يملأ الشاشة كاملة. سنشرح Flexbox في تفصيل في الدرس 6!

📝 التمارين العملية Exercises

تمرين 1: الفرق بين Expo وCLI

اذكر فرقين رئيسيين بين Expo و React Native CLI.

الحل:

1. سهولة الإعداد: Expo لا يحتاج Android Studio، بينما CLI يحتاج إعداداً معقداً
2. التشغيل: Expo يشغّل على جهازك الحقيقي فوراً بـ Expo Go، بينما CLI يحتاج محاكياً

تمرين 2: أمر الإنشاء

اكتب الأمر الكامل لإنشاء مشروع Expo اسمه MyPortfolioApp بالقالب الفارغ.

الحل: 
npx create-expo-app MyPortfolioApp --template blank
تمرين 3: مكونات Expo الثلاثة

اذكر الأجزاء الثلاثة لـ Expo ووظيفة كل منها.

الحل:

Expo SDK: مجموعة مكتبات جاهزة (كاميرا، GPS، إشعارات...)
Expo CLI: أداة سطر أوامر لإنشاء وتشغيل المشاريع
Expo Go: تطبيق على الجهاز لمعاينة التطبيق فوراً

تمرين 4: Fast Refresh

ما هو Fast Refresh ولماذا هو مفيد في تطوير تطبيقات الجوال؟

الحل:

Fast Refresh هو ميزة تُظهر التغييرات في الكود على الجهاز فوراً (خلال ثوانٍ) بدون إعادة تشغيل التطبيق كاملاً، مما يوفّر الوقت ويسرّع التطوير.

تمرين 5: إضافات VS Code

اذكر إضافتين مفيدتين لـ VS Code عند تطوير React Native.

الحل:

React Native Tools: دعم كامل لـ RN مع debugging وتلميحات
ES7+ React/Redux Snippets: اختصارات سريعة (اكتب rnfe + Tab لإنشاء مكوّن كامل)
