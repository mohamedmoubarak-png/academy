.1 ما هو JSX ولماذا نستخدمه؟ What is JSX?

تخيّل أنك تكتب وصفة طبخ — بدل ما تقول "خذ طبق، ضع فيه خضار، ثم اسكب فوقه صلصة" بشكل نثري، ترسم الطبق مباشرة بتصميمه! JSX هو نفس الفكرة: بدل كتابة الكود بشكل نثري، تكتب الواجهة بشكل بصري.

JSX = JavaScript + XML — صيغة كتابة تشبه HTML لكنها تُترجم إلى كود JavaScript عادي.
❌ بدون JSX (صعب القراءة)
React.createElement(
  View, {style: styles.box},
  React.createElement(
    Text, {style: styles.title},
    'مرحباً!'
  ),
  React.createElement(
    Text, null,
    'أهلاً بك'
  )
)
✅ مع JSX (سهل وواضح)
<View style={styles.box}>
  <Text style={styles.title}>
    مرحباً!
  </Text>
  <Text>
    أهلاً بك
  </Text>
</View>
المقارنة اليومية: JSX مثل رسم المبنى بمخطط معماري مباشرةً، بدل وصف كل طوبة في جملة منفصلة. الناتج النهائي واحد لكن الأول أوضح بكثير!

.2 قواعد JSX الأساسية JSX Rules

1️⃣
عنصر جذر واحد دائماً
كل مكوّن يجب أن يُرجع عنصراً واحداً فقط في الجذر. استخدم <View> أو Fragment (<></>) لتجميع العناصر.
❌ خطأ
return (
  <Text>سطر 1</Text>
  <Text>سطر 2</Text>
);
✅ صح
return (
  <>
    <Text>سطر 1</Text>
    <Text>سطر 2</Text>
  </>
);
2️⃣
أغلق كل الوسوم
في JSX لا يوجد وسوم مفتوحة! كل عنصر يجب إغلاقه إما بـ <View></View> أو بنفسه <View />
// ✅ صح — إغلاق ذاتي
<Image source={logo} style={styles.img} />
<TextInput placeholder="اكتب هنا..." />

// ❌ خطأ في JSX (صح في HTML)
// <Image source={logo}>
// <br>  ← لا يوجد br في RN أصلاً!
3️⃣
الأنماط بـ style={} بدل className
في HTML تكتب className="title"، في React Native تكتب style={styles.title}. لا يوجد CSS خارجي في RN.
// HTML/React Web
<div className="header">...</div>

// React Native
<View style={styles.header}>...</View>
<View style={{ padding: 16, margin: 8 }}>...</View>
4️⃣
JavaScript داخل {} في JSX
أي تعبير JavaScript يُكتب داخل أقواس معقوصة {} في JSX.
const name = 'سارة';
const score = 95;

return (
  <View>
    {/* متغير */}
    <Text>مرحباً {name}!</Text>

    {/* حساب */}
    <Text>النتيجة: {score * 10 / 100}</Text>

    {/* شرط مختصر */}
    <Text>{score >= 90 ? 'ممتاز 🏆' : 'جيد 👍'}</Text>

    {/* تعليق داخل JSX */}
    {/* هذه تعليق لا يظهر */}
  </View>
);
تذكّر: التعليقات في JSX تكتب هكذا: {/* تعليق */} وليس // تعليق داخل JSX!

.3 Destructuring — فك التعبئة Destructuring

فكّر في الـ Destructuring مثل فتح صندوق هدية واستخراج كل شيء دفعة واحدة بدل ما تقول "أعطني الجزء الأول، ثم الثاني، ثم الثالث".

📦 Destructuring الـ Objects
// بدون Destructuring (طويل)
function UserCard(props) {
  return (
    <Text>{props.name} — {props.age} سنة</Text>
  );
}

// مع Destructuring (أنظف)
function UserCard({ name, age }) {
  return (
    <Text>{name} — {age} سنة</Text>
  );
}

// مع قيم افتراضية
function UserCard({ name = 'زائر', age = 0 }) {
  return <Text>{name} — {age} سنة</Text>;
}
📋 Destructuring الـ Arrays
// useState يُرجع Array — Destructuring ضروري هنا!
const [count, setCount] = useState(0);
const [name, setName]   = useState('');
const [loading, setLoading] = useState(false);

// Destructuring في الـ imports أيضاً
import { View, Text, StyleSheet, Image } from 'react-native';
// ↑ هذا نفسه Destructuring من Object!
في RN دائماً تستخدم Destructuring: عند استيراد المكونات من react-native، وعند استخدام useState، وعند استقبال props في المكوّن.

.4 Spread Operator مع Props Spread Operator

الـ Spread Operator (...) مثل "نسخ كل محتوى كيس وصبّه في كيس آخر". ثلاث نقاط فقط تختصر كثير من الكود.

📤 Spread في Objects
const baseStyle = { padding: 16, borderRadius: 10 };
const cardStyle = {
  ...baseStyle,           // ← انسخ كل baseStyle
  backgroundColor: 'white',
  shadowOpacity: 0.1,
};
// cardStyle = { padding:16, borderRadius:10, backgroundColor:'white', shadowOpacity:0.1 }

// دمج أنماط متعددة في style
<View style={[styles.base, styles.card, isActive && styles.active]}>
  ...
</View>
🔄 Spread Props
// مكوّن Button مخصّص
function MyButton({ title, color = '#0ea5e9', ...rest }) {
  // ...rest = كل الـ props الباقية (onPress, style, disabled, إلخ)
  return (
    <TouchableOpacity {...rest}>
      <Text style={{ color }}>{title}</Text>
    </TouchableOpacity>
  );
}

// الاستخدام — كل خاصية إضافية تنتقل تلقائياً
<MyButton
  title="اضغط هنا"
  onPress={() => alert('مرحبا!')}
  disabled={false}
  style={{ marginTop: 10 }}
/>
📋 Spread في Arrays
const fruits = ['تفاح', 'برتقال'];
const allFruits = [...fruits, 'موز', 'عنب'];
// ['تفاح', 'برتقال', 'موز', 'عنب']

// إضافة عنصر لقائمة (state) بدون تغيير الأصل
setItems(prev => [...prev, newItem]);

.5 Arrow Functions والمكونات Arrow Functions

Arrow Functions هي طريقة كتابة الدوال بشكل أقصر. في React Native ستراها في كل مكان.

🏹 أنواع تعريف الدوال
// 1. Function Declaration (تقليدي)
export default function App() {
  return <View>...</View>;
}

// 2. Arrow Function Expression (شائع جداً)
const App = () => {
  return <View>...</View>;
};
export default App;

// 3. Arrow Function مختصرة (بدون return)
const Header = () => (
  <Text style={styles.title}>رأس الصفحة</Text>
);
🎯 Arrow Functions في JSX (Inline)
// onPress مع Arrow Function
<TouchableOpacity onPress={() => console.log('ضُغط!')}>
  <Text>اضغط</Text>
</TouchableOpacity>

// مع منطق أكثر
<TouchableOpacity
  onPress={() => {
    setCount(count + 1);
    showToast('تم الضغط!');
  }}
>
  <Text>{count}</Text>
</TouchableOpacity>

// تمرير معامل
{items.map(item => (
  <Text key={item.id} onPress={() => selectItem(item)}>
    {item.name}
  </Text>
))}
متى تستخدم أيهما؟ كلاهما صحيح في RN. الـ Function Declaration أوضح للمكوّنات الرئيسية، والـ Arrow Function أنظف للمكوّنات الصغيرة والدوال المساعدة.

.6 Template Literals في JSX Template Literals

Template Literals (backticks) تحلّ محل تجميع النصوص بعلامة + وتجعل الكود أكثر وضوحاً.

💬 Template Literals الأساسية
const name = 'فهد';
const score = 87;
const grade = 'B+';

// بدون Template Literals (صعب)
const msg1 = 'مرحباً ' + name + '، حصلت على ' + score + ' (' + grade + ')';

// مع Template Literals (أوضح)
const msg2 = `مرحباً ${name}، حصلت على ${score} (${grade})`;

// في JSX
<Text>{`مرحباً ${name}!`}</Text>
🎨 Template Literals في الأنماط
const primaryColor = '#0ea5e9';
const size = 16;

// تكوين الألوان ديناميكياً
const cardBorder = `1px solid ${primaryColor}`;

// رسائل ديناميكية
const greeting = hour < 12
  ? `صباح الخير ${name} 🌅`
  : `مساء الخير ${name} 🌙`;

// URL ديناميكي
const apiUrl = `https://api.example.com/users/${userId}/posts`;

<Image
  source={{ uri: `https://picsum.photos/${size * 10}` }}
  style={{ width: size * 10, height: size * 10 }}
/>
قاعدة: كلما احتجت لدمج متغير داخل نص، استخدم Template Literals بدل + — الكود يصبح أسهل في القراءة والتعديل.

.7 Optional Chaining و Nullish Coalescing Safe Access

هذان المعاملان يحمون تطبيقك من الأعطال عند التعامل مع بيانات قد تكون null أو undefined.

🔗 Optional Chaining (?.) — الوصول الآمن
const user = {
  name: 'أحمد',
  address: { city: 'الرياض' }
};

// بدون ?. — ممكن يعطل لو address = null
const city1 = user.address.city;       // خطر!

// مع ?. — آمن دائماً
const city2 = user?.address?.city;     // 'الرياض' أو undefined
const zip   = user?.address?.zip;      // undefined (بدون خطأ)
const phone = user?.contact?.phone;    // undefined (بدون خطأ)

// في JSX
<Text>{user?.name}</Text>
<Text>{user?.address?.city}</Text>
❓ Nullish Coalescing (??) — القيمة البديلة
// ?? يُعطي القيمة اليسرى إن كانت غير null/undefined
const name  = user?.name  ?? 'زائر';     // 'أحمد' أو 'زائر'
const score = data?.score ?? 0;          // الدرجة أو 0
const img   = user?.avatar ?? 'default.png';

// مقارنة: ?? مقابل ||
const a = 0 || 'افتراضي';    // 'افتراضي' (0 يُعامل كـ false)
const b = 0 ?? 'افتراضي';    // 0 (لأن 0 ليس null/undefined)

// في JSX — عرض "لا توجد بيانات" بدل فراغ
<Text>{user?.bio ?? 'لا توجد نبذة'}</Text>
<Text>{items?.length ?? 0} عنصر</Text>
❌ بدون حماية
// يتعطل لو data = null
<Text>
  {data.user.name}
</Text>
✅ مع حماية
// آمن دائماً
<Text>
  {data?.user?.name ?? 'مجهول'}
</Text>
قاعدة الإبهام: عند جلب بيانات من API أو قاعدة بيانات، استخدم ?. دائماً — فقد تأتي البيانات ناقصة أو null، والتطبيق لا يجب أن يتعطل!

.8 محرر تفاعلي — JSX Sandbox Live Editor

جرّب مفاهيم ES6 في بيئة محاكاة تفاعلية. عدّل البيانات وشاهد النتيجة على شاشة الجوال:

📱 معاينة حية — JSX + ES6 Destructuring + Template 
const user = {
  name: 'سارة محمد',
  role: 'مطورة Frontend',
  stats: { posts: 42, followers: 1280 },
};

export default function App() {
  // Destructuring
  const { name, role, stats } = user;

  // Template Literal + Optional Chaining
  const greeting = `مرحباً، ${name}!`;
  const followers = stats?.followers ?? 0;

  return (
    <View style={styles.card}>
      <Text style={styles.greet}>
        {greeting}
      </Text>
      <Text style={styles.role}>
        {role}
      </Text>
      <View style={styles.row}>
        <Text>📝 {stats.posts}</Text>
        <Text>👥 {followers}</Text>
      </View>
    </View>
  );
}

📝 التمارين العملية Exercises

تمرين 1: إصلاح JSX

الكود التالي فيه 3 أخطاء JSX، ابحث عنها وصحّحها:

<View>
  مرحباً بالعالم
  <Image source={{ uri: 'https://picsum.photos/100' }}>
  <Text style="fontSize:18">أحمد</Text>
</View>
الأخطاء الثلاثة:

1️⃣ النص مباشرة في View بدون <Text>
2️⃣ <Image> لم تُغلق (يجب <Image ... />)
3️⃣ style="fontSize:18" — يجب أن يكون style={{ fontSize: 18 }}

// الكود الصحيح:
<View>
  <Text>مرحباً بالعالم</Text>
  <Image source={{ uri: 'https://picsum.photos/100' }} />
  <Text style={{ fontSize: 18 }}>أحمد</Text>
</View>
تمرين 2: Destructuring

حوّل هذا المكوّن لاستخدام Destructuring في الـ props:

function ProductCard(props) {
  return (
    <View>
      <Text>{props.name}</Text>
      <Text>{props.price} ريال</Text>
      <Text>{props.category}</Text>
    </View>
  );
}
الحل: 
function ProductCard({ name, price, category = 'عام' }) {
  return (
    <View>
      <Text>{name}</Text>
      <Text>{price} ريال</Text>
      <Text>{category}</Text>
    </View>
  );
}
تمرين 3: Template Literals

حوّل هذا الكود لاستخدام Template Literals بدل علامة +:

const user = { name: 'نورة', age: 22, city: 'جدة' };
const msg = 'اسمي ' + user.name + ' عمري ' + user.age + ' من ' + user.city;
الحل: 
const user = { name: 'نورة', age: 22, city: 'جدة' };
// مع Destructuring أيضاً
const { name, age, city } = user;
const msg = `اسمي ${name} عمري ${age} من ${city}`;
تمرين 4: Optional Chaining

اكتب سطر كود يعرض response?.data?.users?.[0]?.name — إن كان موجوداً — وإلا يعرض "لا يوجد مستخدم".

الحل: 
const firstName = response?.data?.users?.[0]?.name ?? 'لا يوجد مستخدم';

// في JSX
<Text>
  {response?.data?.users?.[0]?.name ?? 'لا يوجد مستخدم'}
</Text>
تمرين 5: دمج الكل

اكتب مكوّن WeatherCard يقبل props: city, temp, weather ويستخدم: Destructuring + Template Literals + Optional Chaining + Arrow Function.

الحل: 
const WeatherCard = ({ city = 'غير محدد', temp, weather }) => {
  // Optional Chaining + Nullish Coalescing
  const tempDisplay = temp ?? '--';
  const icon = weather?.icon ?? '🌤️';

  // Template Literal
  const title = `طقس ${city} اليوم`;

  return (
    <View style={styles.card}>
      <Text style={styles.title}>{title}</Text>
      <Text style={styles.temp}>{icon} {tempDisplay}°</Text>
      <Text>{weather?.description ?? 'لا توجد بيانات'}</Text>
    </View>
  );
};

اختبر فهمك لـ JSX و ES6 في React Native!

1يمكن لمكوّن React Native أن يُرجع عنصرين منفصلين في المستوى الأعلى بدون Fragment.
2التعليقات داخل JSX تكتب بهذه الصيغة: {/* تعليق */}
3Optional Chaining (?.) يمنع حدوث خطأ عند الوصول إلى خاصية لكائن قيمته null.
4المعامل ?? و || لهما نفس السلوك تماماً في JavaScript.
5Destructuring يُتيح استخراج قيم من Object أو Array في سطر واحد.
