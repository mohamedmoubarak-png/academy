.1 StyleSheet.create — لماذا وكيف؟ StyleSheet

تخيّل أنك تبني بيتاً — إما تشتري الطوب في كل مرة تحتاجه، أو تجهّزه كله مرة واحدة في المستودع. StyleSheet.create هو المستودع: تُعرّف الأنماط مرة واحدة، ويستخدمها التطبيق بكفاءة عالية.

🎨 StyleSheet.create — الطريقة الصحيحة
import { View, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <View style={styles.card}>
        <Text style={styles.title}>عنوان</Text>
        <Text style={styles.body}>محتوى النص</Text>
      </View>
    </View>
  );
}

// يُعرَّف خارج المكوّن — مرة واحدة عند تحميل الملف
const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f0f9ff',
    padding: 20,
  },
  card: {
    backgroundColor: 'white',
    borderRadius: 16,
    padding: 20,
    elevation: 4,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
  },
  title: { fontSize: 22, fontWeight: 'bold', color: '#0f172a', marginBottom: 8 },
  body:  { fontSize: 15, color: '#64748b', lineHeight: 24 },
});
لماذا خارج المكوّن؟ لو عرّفت StyleSheet داخل الدالة، ستُعاد إنشاؤها في كل render. خارجها: تُنشأ مرة واحدة فقط = أداء أفضل.

.2 Inline Styles مقابل StyleSheet Inline vs StyleSheet

⚠️ Inline (للاختبار فقط)
// تُنشأ object جديد في كل render
<Text style={{
  fontSize: 18,
  color: '#0369a1',
  fontWeight: 'bold',
}}>
  عنوان
</Text>
✅ StyleSheet (المعتاد)
// مُحسَّب مرة واحدة
<Text style={styles.title}>
  عنوان
</Text>

// ...
const styles = StyleSheet.create({
  title: {
    fontSize: 18,
    color: '#0369a1',
    fontWeight: 'bold',
  },
});

دمج أنماط متعددة — Array Style:

🔀 دمج الأنماط
// تمرير Array من الأنماط — الأخير يتغلب
<View style={[styles.card, styles.featured]}>...</View>

// نمط شرطي
<View style={[styles.btn, isActive && styles.btnActive]}>...</View>

// مع Inline للتخصيص السريع
<Text style={[styles.text, { color: accentColor }]}>...</Text>
متى تستخدم Inline؟ فقط عندما تكون القيمة ديناميكية بالكامل (مثل لون يأتي من props أو state). غير ذلك استخدم StyleSheet.

.3 Flexbox — مفهوم الحاوية والعناصر Flexbox Concept

Flexbox في RN يعمل تماماً مثل CSS Flexbox — لكن بفارق واحد مهم: كل View هي Flex Container تلقائياً باتجاه عمودي.

القاعدة الذهبية: في Flexbox يوجد محوران:
• المحور الرئيسي (Main Axis): اتجاه المحتوى — يتحكم فيه flexDirection
• المحور المتقاطع (Cross Axis): العمودي على المحور الرئيسي
flexDirection: 'column' (الافتراضي في RN)

📦 الفرق الرئيسي عن CSS
/* CSS الويب — Flexbox غير مفعّل افتراضياً */
.container { display: flex; flex-direction: row; }

// React Native — كل View هي Flex تلقائياً!
const styles = StyleSheet.create({
  container: {
    // display: 'flex' ← غير ضروري، هو الافتراضي
    flexDirection: 'row',  // فقط إن أردت أفقياً
  },
});

.7 flex — توزيع المساحة flex Property

خاصية flex تُحدد كيف يأخذ العنصر المساحة المتاحة. فكّر فيها كـ "حصص" من المساحة.

📐 flex — توزيع الحصص
// flex: 1 — اشغل كل المساحة المتاحة
container: { flex: 1 }   // الشاشة كلها

// توزيع متساوي بين 3 أقسام
<View style={{ flex: 1, flexDirection: 'row' }}>
  <View style={{ flex: 1, backgroundColor: '#dbeafe' }} />  // 33%
  <View style={{ flex: 1, backgroundColor: '#fce7f3' }} />  // 33%
  <View style={{ flex: 1, backgroundColor: '#d1fae5' }} />  // 33%
</View>

// توزيع غير متساوٍ — 2:1:1
<View style={{ flex: 1, flexDirection: 'row' }}>
  <View style={{ flex: 2, backgroundColor: '#0ea5e9' }} />  // 50%
  <View style={{ flex: 1, backgroundColor: '#7c3aed' }} />  // 25%
  <View style={{ flex: 1, backgroundColor: '#f43f5e' }} />  // 25%
</View>

📝 التمارين العملية Exercises

تمرين 1: توسيط كامل

اكتب StyleSheet لـ container يُوسّط محتواه أفقياً وعمودياً، مع خلفية #f0f9ff وتملأ الشاشة كلها.

الحل: 
const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f0f9ff',
  },
});
تمرين 2: شريط تنقل

اكتب View تُشبه Navbar: صورة شخصية على اليسار، عنوان في المنتصف، أيقونة إشعارات على اليمين.

الحل: 
<View style={styles.navbar}>
  <Image source={avatar} style={styles.avatar} />
  <Text style={styles.navTitle}>الرئيسية</Text>
  <TouchableOpacity>
    <Text style={{ fontSize: 22 }}>🔔</Text>
  </TouchableOpacity>
</View>

navbar: {
  flexDirection: 'row',
  justifyContent: 'space-between',
  alignItems: 'center',
  padding: 16,
  backgroundColor: 'white',
},
تمرين 3: توزيع flex

قسّم الشاشة أفقياً: الجزء الأيسر يأخذ 30% والأيمن 70%. استخدم flex بدون نسب مئوية.

الحل: 
<View style={{ flex: 1, flexDirection: 'row' }}>
  <View style={{ flex: 3, backgroundColor: '#1e293b' }} />
  {/* 3/(3+7) = 30% */}
  <View style={{ flex: 7, backgroundColor: '#f0f9ff' }} />
  {/* 7/(3+7) = 70% */}
</View>
تمرين 4: نمط شرطي

اكتب زر يتغير لونه من أزرق إلى أخضر عندما تكون isSelected = true. استخدم Array Style.

الحل: 
const [isSelected, setIsSelected] = useState(false);

<TouchableOpacity
  style={[styles.btn, isSelected && styles.btnSelected]}
  onPress={() => setIsSelected(!isSelected)}
>
  <Text style={styles.btnText}>
    {isSelected ? 'محدد ✓' : 'اضغط'}
  </Text>
</TouchableOpacity>

btn:         { backgroundColor: '#0ea5e9', padding: 12, borderRadius: 8 },
btnSelected: { backgroundColor: '#16a34a' },
btnText:     { color: 'white', fontWeight: 'bold' },
تمرين 5: بطاقة منتج

صمّم بطاقة منتج تحتوي: صورة في الأعلى، ثم صف أفقي فيه الاسم (flex:1) والسعر (ثابت)، ثم زر "أضف للسلة" في الأسفل.

الحل: 
<View style={styles.card}>
  <Image source={{ uri: imgUrl }} style={styles.img} resizeMode="cover" />
  <View style={styles.row}>
    <Text style={[styles.name, { flex: 1 }]} numberOfLines={1}>اسم المنتج</Text>
    <Text style={styles.price}>99 ريال</Text>
  </View>
  <TouchableOpacity style={styles.btn}>
    <Text style={styles.btnText}>🛒 أضف للسلة</Text>
  </TouchableOpacity>
</View>

card: { backgroundColor:'white', borderRadius:14, overflow:'hidden', elevation:3 },
img:  { width:'100%', height:140 },
row:  { flexDirection:'row', alignItems:'center', padding:10, gap:8 },
btn:  { backgroundColor:'#0ea5e9', margin:10, padding:10, borderRadius:10, alignItems:'center' },
