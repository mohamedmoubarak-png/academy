.1 المناطق غير الآمنة Unsafe Zones

الجوال الحديث ليس مجرد شاشة مستطيلة نظيفة — هناك شريط الحالة فوق، وزر الهوم أو شريط الإيماءات أسفل، والـ notch في الأجهزة الجديدة. لو وضعت محتواك بدون اعتبار، سيختفي خلف هذه العناصر!

ما هي المناطق؟

المنطقة غير الآمنة خلف شريط الحالة أو الـ notch أو شريط الإيماءات — المحتوى هنا مقطوع أو مخفي.
المنطقة الآمنة (Safe Area) المساحة المرئية الكاملة التي يمكن للمستخدم رؤيتها والتفاعل معها.
تختلف بحسب الجهاز iPhone بـ notch: 44px فوق. iPhone بـ Dynamic Island: 59px. Android: يتفاوت.
المشكلة الشائعة: تضع زر أو نص مهم في أسفل الشاشة فيختفي خلف شريط الإيماءات على iPhone X+. الحل: SafeAreaView.

.2 SafeAreaView — الحل الأساسي SafeAreaView

بدل View العادية، استخدم SafeAreaView كحاوية الشاشة الرئيسية — تُضيف padding تلقائياً لتجنب المناطق غير الآمنة.

🛡️ SafeAreaView الأساسية
import { SafeAreaView, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    {/* استبدل View الخارجية بـ SafeAreaView */}
    <SafeAreaView style={styles.container}>
      <Text>هذا النص لن يختفي خلف الـ notch ✅</Text>
    </SafeAreaView>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#f0f9ff',
    // SafeAreaView تُضيف الـ padding تلقائياً
  },
});
تنبيه: SafeAreaView المدمجة في react-native تعمل على iOS فقط. للأندرويد أيضاً استخدم مكتبة react-native-safe-area-context:
📦 react-native-safe-area-context (الأفضل)
// التثبيت
npx expo install react-native-safe-area-context

// الاستخدام — يعمل على iOS و Android
import {
  SafeAreaProvider,
  SafeAreaView,
} from 'react-native-safe-area-context';

// في App.js الرئيسي
export default function App() {
  return (
    <SafeAreaProvider>
      <SafeAreaView style={{ flex: 1 }}>
        {/* كل محتوى تطبيقك هنا */}
      </SafeAreaView>
    </SafeAreaProvider>
  );
}
أين تضعها؟ SafeAreaView تُحاط بالشاشة الكاملة — أي المكوّن الأعلى مستوى في كل شاشة، وليس داخل البطاقات أو القوائم.

.3 useSafeAreaInsets — التحكم الدقيق useSafeAreaInsets

أحياناً تحتاج قيم الـ insets نفسها — مثلاً لتضيف padding يدوياً لعنصر معين فقط.

📐 useSafeAreaInsets
import { useSafeAreaInsets } from 'react-native-safe-area-context';

function BottomBar() {
  const insets = useSafeAreaInsets();

  return (
    <View
      style={{
        position: 'absolute',
        bottom: 0,
        left: 0, right: 0,
        backgroundColor: 'white',
        paddingBottom: insets.bottom,  // 34px على iPhone X
        paddingTop: 12,
        borderTopWidth: 1,
        borderColor: '#e2e8f0',
      }}
    >
      {/* أيقونات التنقل */}
    </View>
  );
}

// القيم المتاحة:
// insets.top    — المسافة من أعلى (notch/status bar)
// insets.bottom — المسافة من أسفل (home indicator)
// insets.left   — المسافة من اليسار (landscape mode)
// insets.right  — المسافة من اليمين (landscape mode)

📊 StatusBar الكامل
import { StatusBar } from 'react-native';
// أو من expo-status-bar (أفضل مع Expo)
import { StatusBar } from 'expo-status-bar';

export default function App() {
  return (
    <SafeAreaView>
      {/* expo-status-bar */}
      <StatusBar
        style="dark"          // 'auto' | 'light' | 'dark'
        backgroundColor="#f0f9ff"  // Android فقط
        translucent={false}
      />

      {/* react-native StatusBar */}
      <StatusBar
        barStyle="dark-content"   // 'default' | 'light-content' | 'dark-content'
        backgroundColor="white"   // Android
        hidden={false}            // إخفاء الشريط كلياً
      />
    </SafeAreaView>
  );
}
Expo vs React Native StatusBar: مع Expo استخدم expo-status-bar لأنها أبسط وتتعامل مع الاختلافات تلقائياً. استخدم react-native StatusBar فقط لو تحتاج تحكماً أدق.

.5 KeyboardAvoidingView — تجنب لوحة المفاتيح KeyboardAvoidingView

عند فتح لوحة المفاتيح على الجوال، تُغطي جزءاً من الشاشة — قد يختفي حقل الإدخال أو الزر! KeyboardAvoidingView يُحرّك المحتوى للأعلى تلقائياً.

كيف يعمل؟

يستمع لحدث ظهور/اختفاء لوحة المفاتيح، ثم يُحرّك المحتوى بـ padding أو height أو position لإبقاء حقل الإدخال ظاهراً.

⌨️ KeyboardAvoidingView — الاستخدام الكامل
import {
  KeyboardAvoidingView, Platform,
  ScrollView, TextInput, TouchableOpacity, Text,
} from 'react-native';

export default function LoginScreen() {
  return (
    <SafeAreaView style={{ flex: 1 }}>
      <KeyboardAvoidingView
        style={{ flex: 1 }}
        behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
        keyboardVerticalOffset={0}    // ضبط إضافي
      >
        <ScrollView
          contentContainerStyle={{ flexGrow: 1 }}
          keyboardShouldPersistTaps="handled"
        >
          <TextInput
            placeholder="البريد الإلكتروني"
            style={styles.input}
          />
          <TextInput
            placeholder="كلمة المرور"
            secureTextEntry
            style={styles.input}
          />
          <TouchableOpacity style={styles.btn}>
            <Text style={styles.btnText}>تسجيل الدخول</Text>
          </TouchableOpacity>
        </ScrollView>
      </KeyboardAvoidingView>
    </SafeAreaView>
  );
}
📦
'padding'
يُضيف padding-bottom. مثالي لـ iOS.
📏
'height'
يُقلص الارتفاع. مفيد على Android.
📌
'position'
يرفع العنصر بـ position. نادراً.
القاعدة: استخدم behavior={Platform.OS === 'ios' ? 'padding' : 'height'} دائماً — هذا الأنسب لكلا النظامين.

.6 Keyboard API — التحكم البرمجي Keyboard API

⌨️ Keyboard API
import { Keyboard, TouchableWithoutFeedback, View } from 'react-native';

// 1. إغلاق لوحة المفاتيح برمجياً
Keyboard.dismiss();

// 2. إغلاقها عند الضغط خارج الحقل (الشائع جداً)
<TouchableWithoutFeedback onPress={Keyboard.dismiss}>
  <View style={{ flex: 1 }}>
    <TextInput ... />
  </View>
</TouchableWithoutFeedback>

// 3. الاستماع لأحداث لوحة المفاتيح
import { useEffect } from 'react';

useEffect(() => {
  const showSub = Keyboard.addListener('keyboardDidShow', (e) => {
    console.log('ارتفاع الكيبورد:', e.endCoordinates.height);
  });
  const hideSub = Keyboard.addListener('keyboardDidHide', () => {
    console.log('الكيبورد اختفى');
  });

  return () => { showSub.remove(); hideSub.remove(); };
}, []);

.7 تجميع الثلاثة في تطبيق Putting It Together

هذا هو الهيكل المعتاد لشاشة تسجيل دخول احترافية تجمع الثلاثة:

🏗️ هيكل شاشة Login كاملة
import { SafeAreaView, KeyboardAvoidingView,
  ScrollView, Platform, StatusBar,
  TouchableWithoutFeedback, Keyboard,
  View, Text, TextInput, TouchableOpacity,
  StyleSheet } from 'react-native';
import { StatusBar as ExpoStatusBar } from 'expo-status-bar';

export default function LoginScreen() {
  return (
    {/* 1️⃣ SafeAreaView — المنطقة الآمنة */}
    <SafeAreaView style={s.safe}>

      {/* 2️⃣ StatusBar — شريط الحالة */}
      <ExpoStatusBar style="dark" />

      {/* 3️⃣ KeyboardAvoidingView — تجنب الكيبورد */}
      <KeyboardAvoidingView
        style={{ flex: 1 }}
        behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
      >
        {/* إغلاق الكيبورد عند الضغط خارج الحقل */}
        <TouchableWithoutFeedback onPress={Keyboard.dismiss}>
          <ScrollView
            contentContainerStyle={s.scroll}
            keyboardShouldPersistTaps="handled"
          >
            <Text style={s.title}>أهلاً بك 👋</Text>
            <TextInput style={s.input} placeholder="الإيميل" />
            <TextInput style={s.input} placeholder="كلمة المرور"
              secureTextEntry />
            <TouchableOpacity style={s.btn}>
              <Text style={s.btnText}>دخول</Text>
            </TouchableOpacity>
          </ScrollView>
        </TouchableWithoutFeedback>
      </KeyboardAvoidingView>

    </SafeAreaView>
  );
}

📝 التمارين العملية Exercises

تمرين 1: تحديد المشكلة

لاحظ المستخدمون أن زر "تأكيد" يختفي خلف شريط الإيماءات في iPhone X. ما الحل وكيف تطبّقه؟

الحل:

استخدم SafeAreaView كحاوية الشاشة، أو أضف paddingBottom: insets.bottom بـ useSafeAreaInsets على الـ View التي تحتوي الزر.

import { useSafeAreaInsets } from 'react-native-safe-area-context';

function BottomButton() {
  const { bottom } = useSafeAreaInsets();
  return (
    <View style={{ paddingBottom: bottom + 16 }}>
      <TouchableOpacity style={styles.btn}>
        <Text>تأكيد</Text>
      </TouchableOpacity>
    </View>
  );
}
تمرين 2: ألوان StatusBar

لو شاشتك خلفيتها داكنة (#0f172a)، ما الـ style الصحيح لـ StatusBar؟ ولو الخلفية بيضاء؟

الحل:

• خلفية داكنة → style="light" (نص أبيض يظهر على الداكن)
• خلفية بيضاء → style="dark" (نص داكن يظهر على الفاتح)

// خلفية داكنة
<StatusBar style="light" />

// خلفية فاتحة
<StatusBar style="dark" />
تمرين 3: KeyboardAvoidingView

اكتب هيكل شاشة تسجيل تحتوي: SafeAreaView → KeyboardAvoidingView → ScrollView → حقلا إدخال + زر. مع behavior الصحيح لكل منصة.

الحل: 
<SafeAreaView style={{ flex: 1 }}>
  <KeyboardAvoidingView
    style={{ flex: 1 }}
    behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
  >
    <ScrollView

</SafeAreaView>

تمرين 5: Tab Bar آمنة

لديك Tab Bar ثابتة في أسفل الشاشة (position: 'absolute', bottom: 0). كيف تجعلها آمنة على iPhone X؟

الحل: 
import { useSafeAreaInsets } from 'react-native-safe-area-context';

function TabBar() {
  const { bottom } = useSafeAreaInsets();

  return (
    <View style={{
      position: 'absolute',
      bottom: 0, left: 0, right: 0,
      paddingBottom: bottom,     // 34px على iPhone X+
      backgroundColor: 'white',
      flexDirection: 'row',
      borderTopWidth: 1,
      borderColor: '#e2e8f0',
    }}>
      ...
    </View>
  );
}

اختبر فهمك لـ SafeArea, StatusBar, و KeyboardAvoidingView!

1SafeAreaView المدمجة في react-native تعمل بشكل كامل على iOS و Android.
2KeyboardAvoidingView يحتاج behavior مختلف على iOS وAndroid.
3Keyboard.dismiss() يُغلق لوحة المفاتيح برمجياً.
4style="dark" في StatusBar يعني أن الخلفية ستكون داكنة.
5useSafeAreaInsets يُعطي قيمة insets.bottom للمسافة من أسفل الشاشة.
