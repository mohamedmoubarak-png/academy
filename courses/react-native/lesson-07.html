.2 الوحدات في React Native Units in RN

في React Native، الأرقام بدون وحدة هي density-independent pixels (dp) — وحدات مستقلة عن كثافة الشاشة. لا يوجد px، rem، أو em!

أرقام (dp)
القيمة الافتراضية. width: 100 = 100dp.
متى: أبعاد ثابتة (أيقونات، حدود)
نسب مئوية '%'
كنص داخل علامتي تنصيص. width: '50%'.
متى: نسبة من الأب المباشر
flex: number
حصة من المساحة. flex: 1 = اشغل الباقي.
متى: الأشيع للتخطيطات المرنة
Dimensions
قيم محسوبة من حجم الشاشة الفعلي.
متى: تخطيطات معقدة (50% - 20px)
📏 أمثلة على الوحدات
// 1️⃣ أرقام = dp (الأشيع)
{ width: 100, height: 100, padding: 16 }

// 2️⃣ نسب مئوية كـ string
{ width: '50%', height: '100%' }

// 3️⃣ flex — حصص من المساحة
{ flex: 1 }      // 100% من الباقي
{ flex: 2 }      // ضعف العنصر الذي flex: 1

// 4️⃣ Dimensions — قيم ديناميكية
const screenWidth = Dimensions.get('window').width;
{ width: screenWidth * 0.9 }   // 90% بطريقة برمجية
ما هي dp؟ وحدة تتكيّف مع كثافة الشاشة — حتى يبدو الزر بنفس الحجم البصري على كل الأجهزة، سواء كانت شاشتها كثيفة أو عادية.

.3 Dimensions API Dimensions API

للحصول على الأبعاد الحقيقية للشاشة، نستخدم Dimensions من react-native:

Dimensions.get('window')
جلب أبعاد النافذة الحالية
يُرجع object بـ { width, height, scale, fontScale }
📐 استخدام Dimensions
import { View, Text, Dimensions, StyleSheet } from 'react-native';

// جلب الأبعاد مرة واحدة
const { width, height } = Dimensions.get('window');

export default function App() {
  return (
    <View style={styles.container}>
      <Text>عرض الشاشة: {width}px</Text>
      <Text>طول الشاشة: {height}px</Text>

      {/* بطاقة بـ 90% من العرض */}
      <View style={{ width: width * 0.9, height: 200 }}>
        ...
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center' },
});

الفرق بين 'window' و 'screen':

القيمةالمعنىمتى تستخدمها
'window'المساحة المتاحة لتطبيقك (بدون شريط الحالة)الاستخدام الشائع (99%)
'screen'كامل الشاشة الفيزيائيةنادراً — قياس شاشة الجهاز كاملة
عيب Dimensions.get(): يُحسب مرة واحدة فقط — لا يتحدث عند دوران الجهاز أو تغيير حجم النافذة. للحل: استخدم useWindowDimensions في القسم التالي.

.4 useWindowDimensions — الطريقة الحديثة Modern Way

الـ Hook useWindowDimensions هو الحل الموصى به — يُعيد تشغيل المكوّن تلقائياً عند تغيير الأبعاد.

useWindowDimensions()
Hook متجاوب مع تغييرات الشاشة
يستمع للتغييرات تلقائياً (دوران الشاشة، تغيير حجم النافذة، الـ Split Screen).
⚡ useWindowDimensions — الموصى به
import { View, Text, useWindowDimensions } from 'react-native';

export default function App() {
  const { width, height } = useWindowDimensions();

  // تخطيط متكيّف — صف للشاشات الكبيرة، عمود للصغيرة
  const isLandscape = width > height;

  return (
    <View style={{
      flex: 1,
      flexDirection: isLandscape ? 'row' : 'column',
    }}>
      <View style={{ flex: 1, backgroundColor: '#0ea5e9' }} />
      <View style={{ flex: 2, backgroundColor: '#7c3aed' }} />
    </View>
  );
}
❌ Dimensions.get()
قيمة ثابتة — لا تتحدث عند دوران الشاشة
✅ useWindowDimensions()
يتحدث تلقائياً — مثالي للتطبيقات الحديثة
القاعدة: في المكوّنات الحديثة، استخدم useWindowDimensions دائماً. اترك Dimensions.get فقط للاستخدامات خارج المكوّنات (constants).

.5 النسب المئوية والتصميم النسبي Percentage Sizing

النسب المئوية في RN تختلف قليلاً عن CSS — يجب كتابتها كـ string:

📊 النسب المئوية
// ✅ صحيح — كـ string
{ width: '80%', height: '50%' }

// ❌ خطأ — كرقم
{ width: 80% }     // Syntax Error

// ❌ خطأ شائع — بدون علامتي تنصيص
{ width: "80" }    // تُفسَّر كـ 80dp وليس 80%
🎯 مثال عملي — بطاقة متجاوبة
// بطاقة تشغل 90% من العرض على كل الأجهزة
const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
  },
  card: {
    width: '90%',          // نسبة من الأب
    maxWidth: 500,         // حد أقصى للأجهزة الكبيرة
    padding: 20,
    backgroundColor: 'white',
    borderRadius: 12,
  },
});
تنبيه مهم: النسب تُحسب من الأب المباشر، وليس من الشاشة. لو الأب عرضه 200، فـ '50%' = 100.

.6 PixelRatio — كثافة الشاشة PixelRatio

الشاشات الحديثة تختلف في كثافة البكسل — شاشة Retina تعرض ضعف البكسلات في نفس المساحة. PixelRatio يساعدك في التعامل مع هذا.

🔍 PixelRatio API
import { PixelRatio } from 'react-native';

// 1. الحصول على كثافة الشاشة
const ratio = PixelRatio.get();
// iPhone عادي: 2, Retina: 3, Android متوسط: 2-3

// 2. حجم الخط النسبي (يحترم إعدادات المستخدم)
const fontScale = PixelRatio.getFontScale();

// 3. تحويل dp إلى بكسلات حقيقية
const pixels = PixelRatio.getPixelSizeForLayoutSize(100);
// 100dp × 3 = 300 بكسل حقيقي على Retina

// 4. تقريب لأقرب بكسل (لتجنب التشويش)
const roundedSize = PixelRatio.roundToNearestPixel(8.4);
متى تستخدمه؟ غالباً لا تحتاجه! React Native تتعامل مع الكثافة تلقائياً. تستخدمه فقط للحالات الخاصة كالرسومات المخصصة (Canvas, SVG) أو الصور عالية الدقة.

.7 Platform — التفريق بين iOS و Android Platform API

أحياناً تحتاج تنسيقاً مختلفاً لكل نظام — Platform API يساعدك:

🍎
iOS
  • ظلال بـ shadowColor + shadowOffset
  • SafeAreaView مهمة (notch + bezel)
  • خط San Francisco افتراضي
  • التنقل من اليسار للخلف
🤖
Android
  • ظلال بـ elevation فقط
  • StatusBar أكثر تخصيصاً
  • خط Roboto افتراضي
  • زر Back فيزيائي/برمجي
📱 استخدام Platform
import { Platform, StyleSheet } from 'react-native';

// 1. التحقق من النظام
if (Platform.OS === 'ios') {
  console.log('تطبيق على iPhone');
}

// 2. قيمة مختلفة لكل نظام
const headerHeight = Platform.OS === 'ios' ? 88 : 64;

// 3. الطريقة الأنظف — Platform.select
const styles = StyleSheet.create({
  card: {
    backgroundColor: 'white',
    borderRadius: 12,
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.1,
        shadowRadius: 4,
      },
      android: {
        elevation: 4,
      },
    }),
  },
});

// 4. التحقق من إصدار النظام
if (Platform.Version >= 14) {
  // ميزة iOS 14+
}
قاعدة عملية: الظلال هي أكثر مكان تحتاج فيه Platform.select — لأن iOS يستخدم shadow* و Android يستخدم elevation.

.8 معاينة حية — تطبيق متجاوب Live Preview

كود حقيقي يجمع كل ما تعلّمته — تطبيق يتكيّف مع حجم الشاشة:

📱 معاينة حية — Responsive Card useWindowDimensions + Platform 
import {
  View, Text, Platform,
  useWindowDimensions,
} from 'react-native';

export default function App() {
  const { width } = useWindowDimensions();

  // عرض ديناميكي — 90% بحد أقصى 500
  const cardW = Math.min(width * 0.9, 500);

  // عمودين على iPad، عمود واحد على الجوال
  const isTablet = width > 600;

  return (
    <View style={s.screen}>
      <View
        style={[
          s.card,
          { width: cardW },
        ]}
      >
        <Text style={s.title}>
          {isTablet ? 'iPad 📟' : 'جوال 📱'}
        </Text>
        <Text style={s.info}>
          العرض: {width}dp
        </Text>
        <Text style={s.info}>
          النظام: {Platform.OS}
        </Text>
      </View>
    </View>
  );
}

📝 التمارين العملية Exercises

تمرين 1: التفريق بين dp و %

أيهما أفضل لكل حالة:
أ) صورة شعار التطبيق بحجم 80
ب) بطاقة منتج تشغل 90% من العرض
ج) حد دائرة بـ 2 بكسل

الحل:

أ) 80 (dp) — لأن الشعار حجم ثابت يجب أن يظهر بنفس الحجم البصري
ب) '90%' — لأنها نسبة من الأب وتتكيف مع كل الأجهزة
ج) 2 (dp) — الحدود الرفيعة دائماً بأرقام صغيرة

تمرين 2: useWindowDimensions

اكتب مكوّن يعرض رسالة مختلفة بناءً على عرض الشاشة:
• أقل من 400: "جوال صغير 📱"
• 400-700: "جوال كبير 📱"
• أكثر من 700: "تابلت 📟"

الحل: 
import { Text, useWindowDimensions } from 'react-native';

function DeviceLabel() {
  const { width } = useWindowDimensions();

  let label;
  if (width < 400) label = 'جوال صغير 📱';
  else if (width < 700) label = 'جوال كبير 📱';
  else label = 'تابلت 📟';

  return <Text>{label}</Text>;
}
تمرين 3: Platform.select

اكتب نمط card له ظل: يستخدم shadow* على iOS و elevation على Android.

الحل: 
import { Platform, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  card: {
    backgroundColor: 'white',
    borderRadius: 12,
    padding: 16,
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.15,
        shadowRadius: 6,
      },
      android: {
        elevation: 5,
      },
    }),
  },
});
تمرين 4: بطاقة بحد أقصى

اكتب نمط لبطاقة عرضها 90% من الشاشة لكن لا يتجاوز 600dp (مهم للأجهزة الكبيرة).

الحل: 
// الطريقة 1: CSS فقط
card: {
  width: '90%',
  maxWidth: 600,
  alignSelf: 'center',
}

// الطريقة 2: بالحساب
const { width } = useWindowDimensions();
const cardWidth = Math.min(width * 0.9, 600);

<View style={{ width: cardWidth }}> ... </View>
تمرين 5: تخطيط متجاوب

اكتب مكوّن يعرض عمودين (flexDirection: 'row') لو الشاشة أكبر من 600dp، وعمود واحد (column) لو أقل.

الحل: 
function ResponsiveLayout() {
  const { width } = useWindowDimensions();
  const isWide = width >= 600;

  return (
    <View style={{
      flex: 1,
      flexDirection: isWide ? 'row' : 'column',
    }}>
      <View style={{ flex: 1, backgroundColor: '#0ea5e9' }} />
      <View style={{ flex: 1, backgroundColor: '#7c3aed' }} />
    </View>
  );
}
