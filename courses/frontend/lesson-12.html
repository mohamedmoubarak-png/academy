.1 ما هو التصميم المتجاوب؟ Responsive Design

التصميم المتجاوب (Responsive Design) يعني إن موقعك يبدو جيداً ويعمل بشكل صحيح على كل الأجهزة — موبايل، تابلت، لاب توب، شاشة كبيرة — بدون ما تعمل نسخة منفصلة لكل جهاز.

💧 تخيل الماء في إناء:

الماء يأخذ شكل أي إناء تضعه فيه. موقعك المتجاوب هكذا بالضبط — يأخذ شكل الشاشة التي يُعرض عليها سواء كانت صغيرة أو كبيرة.

لماذا مهم جداً؟ أكثر من 60% من تصفح الإنترنت الآن يأتي من الموبايل. موقع غير متجاوب = تجربة سيئة = زوار يغادرون.
ثلاثة أعمدة التصميم المتجاوب:
العمودالأداةالدور
شبكة مرنةFlexbox / Gridالتخطيط يتكيف مع العرض
صور مرنةmax-width: 100%الصور لا تفيض عن حاويتها
Media Queries@mediaتغيير التنسيق حسب حجم الشاشة

.2 Viewport وعلامة meta Viewport Meta Tag

أول خطوة في أي موقع متجاوب هي إضافة هذا السطر في <head>:

📱 meta viewport — ضروري دائماً
<meta name="viewport" content="width=device-width, initial-scale=1.0">
ماذا تعني هذه الخصائص؟
  • width=device-width: اجعل عرض الصفحة يساوي عرض شاشة الجهاز
  • initial-scale=1.0: لا تكبّر أو تصغّر الصفحة عند فتحها — مقياس 1:1
بدون هذا السطر: المتصفح على الموبايل يفترض إن الصفحة بعرض 980px ويصغّرها — والنتيجة نصوص صغيرة جداً وتصميم مكسور.

.3 Media Queries @media Rules

Media Queries هي قواعد CSS تُطبَّق فقط لما تتحقق شرط معين — زي عرض الشاشة.

📐 الشكل العام
/* تُطبَّق لما العرض أقل من أو يساوي 768px */
@media (max-width: 768px) {
  .container {
    flex-direction: column;
  }
}

/* تُطبَّق لما العرض أكبر من 1024px */
@media (min-width: 1024px) {
  .sidebar {
    display: block;
  }
}
أنواع الشروط في Media Queries:
🔍 أشهر الشروط
/* عرض الشاشة */
@media (max-width: 600px)  { /* موبايل */ }
@media (min-width: 768px)  { /* تابلت وأكبر */ }
@media (min-width: 1024px) { /* لاب توب وأكبر */ }

/* نطاق بين قيمتين */
@media (min-width: 768px) and (max-width: 1023px) {
  /* تابلت فقط */
}

/* اتجاه الجهاز */
@media (orientation: landscape) { /* أفقي */ }
@media (orientation: portrait)  { /* عمودي */ }
📌 مثال عملي: قائمة تنقل 
/* الافتراضي: قائمة أفقية */
nav {
  display: flex;
  gap: 20px;
}

/* على الموبايل: قائمة عمودية */
@media (max-width: 600px) {
  nav {
    flex-direction: column;
    gap: 8px;
  }
}

.5 Mobile-First Mobile-First Approach

Mobile-First معناها تكتب CSS الموبايل أولاً كـ default، ثم تضيف min-width media queries للشاشات الأكبر. هذا هو الأسلوب المعتمد في كل frameworks حديثة.

Desktop-First (القديم — تجنبه):
❌ Desktop-First
/* الافتراضي للديسكتوب */
.cards { grid-template-columns: repeat(3, 1fr); }

/* تصغّر للموبايل */
@media (max-width: 768px) {
  .cards { grid-template-columns: 1fr; }
}
Mobile-First (الصواب):
✅ Mobile-First
/* الافتراضي للموبايل */
.cards { grid-template-columns: 1fr; }

/* تكبّر للتابلت */
@media (min-width: 768px) {
  .cards { grid-template-columns: repeat(2, 1fr); }
}

/* تكبّر أكثر للديسكتوب */
@media (min-width: 1024px) {
  .cards { grid-template-columns: repeat(3, 1fr); }
}

.6 وحدات مرنة Flexible Units

بدل استخدام px الثابتة في كل مكان، هناك وحدات مرنة تجعل التصميم أكثر تكيّفاً.

الوحدةتعنيمتى تستخدمها؟
%نسبة من الحاوي الأبالعرض: width: 80%
vwنسبة من عرض الشاشةعناصر تملأ الشاشة: width: 100vw
vhنسبة من ارتفاع الشاشةHero sections: height: 100vh
remمضاعف حجم خط الجذر (root font)الخطوط والمسافات: font-size: 1.2rem
emمضاعف حجم خط الأبpadding/margin نسبي للخط
clamp()قيمة بين حد أدنى وأعلىخطوط مرنة بدون media queries
🔢 أمثلة عملية
/* خط يتكيف تلقائياً بين 16px و 32px */
h1 { font-size: clamp(16px, 4vw, 32px); }

/* حاوية بعرض 90% بحد أقصى 1200px */
.container {
  width: 90%;
  max-width: 1200px;
  margin: 0 auto;
}

/* Hero يملأ الشاشة كاملاً */
.hero {
  width: 100vw;
  height: 100vh;
}
القاعدة الذهبية للخطوط: استخدم rem دائماً. الافتراضي في المتصفح هو 16px، فـ 1rem = 16px، و 1.5rem = 24px.

.7 صور متجاوبة Responsive Images

الصور هي أكثر عنصر يسبب مشاكل على الموبايل. ثلاث قواعد تحل 95% من المشاكل:

🖼️ القاعدة الأساسية
/* كل الصور متجاوبة */
img {
  max-width: 100%;   /* لا تفيض عن الحاوي */
  height: auto;      /* حافظ على النسبة */
  display: block;     /* أزل المساحة السفلية */
}
🎨 صورة خلفية متجاوبة
.hero {
  background-image: url('hero.jpg');
  background-size: cover;      /* تغطي المساحة كاملاً */
  background-position: center; /* توسيط
  height: 400px;
}
نصيحة: استخدم object-fit للصور في بطاقات
📌 object-fit
.card-img {
  width: 100%;
  height: 200px;
  object-fit: cover;     /* اقطع بدون تشويه */
  object-position: center;
}

📝 التمارين العملية Exercises

تمرين 1: أضف meta viewport

اكتب السطر الصحيح الذي يجب إضافته في <head> لجعل أي موقع متجاوباً.

الحل: 
<meta name="viewport" content="width=device-width, initial-scale=1.0">
تمرين 2: إخفاء عنصر على الموبايل

اكتب media query تخفي عنصر .sidebar على الشاشات الأصغر من 768px.

الحل: 
@media (max-width: 767px) {
  .sidebar {
    display: none;
  }
}
تمرين 3: خط متجاوب بـ clamp

اكتب CSS لعنوان h1 يكون حجم خطه: لا يقل عن 20px، ولا يزيد عن 48px، ويتكيف مع عرض الشاشة.

الحل: 
h1 {
  font-size: clamp(20px, 5vw, 48px);
}
تمرين 4: تخطيط Mobile-First

اكتب CSS بأسلوب Mobile-First لـ .grid: عمود واحد على الموبايل، عمودان عند 640px فأكثر، ثلاثة عند 1024px فأكثر.

الحل: 
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 640px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}
تمرين 5: حاوية متمركزة

اكتب CSS لحاوية .container بعرض 90%، بحد أقصى 1200px، ومتمركزة أفقياً.

الحل: 
.container {
  width: 90%;
  max-width: 1200px;
  margin: 0 auto;
}
