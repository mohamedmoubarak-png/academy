.1 لماذا نقسّم الكود؟ Why Modules?

لما يكبر مشروعك، كتابة كل الكود في ملف واحد تصبح كارثة. الـ Modules تحل هذه المشكلة.

📚 تخيل مكتبة:

الكتب مقسّمة حسب الموضوع — علوم، رياضيات، أدب... كل قسم مستقل وسهل الإيجاد. Modules هكذا — كل ملف مسؤول عن وظيفة محددة.

بدون Modulesمع Modules
ملف واحد ضخمملفات صغيرة متخصصة
صعب الصيانةسهل التعديل والاختبار
تعارض أسماء المتغيراتكل module له نطاقه الخاص
لا إعادة استخدامimport في أي مكان

.2 ES Modules import / export

📤 math.js — التصدير
// math.js
export const PI = 3.14159;

export function add(a, b)      { return a + b; }
export function multiply(a, b) { return a * b; }
export const   square = n => n ** 2;
📥 app.js — الاستيراد
// استيراد ما تحتاجه فقط
import { add, PI } from './math.js';

console.log(add(3, 4));  // 7
console.log(PI);       // 3.14159

// استيراد بأسماء مختلفة
import { multiply as mul } from './math.js';
console.log(mul(3, 4));  // 12

// استيراد كل شيء
import * as Math from './math.js';
Math.square(5);  // 25
في HTML: أضف type="module" للسكريبت:
<script type="module" src="app.js"></script>
Modules تعمل مع server فقط (لا تعمل بفتح الملف مباشرة بـ file://).

.3 Named vs Default Export Types

🏷️ Named Export — أسماء محددة
// utils.js
export const VERSION  = '1.0';
export function format(d) { ... }
export function validate(v) { ... }

// الاستيراد: الأقواس {} إجبارية
import { VERSION, format } from './utils.js';
⭐ Default Export — تصدير افتراضي
// Button.js — مكوّن واحد رئيسي
export default function Button({ text, onClick }) {
  const btn = document.createElement('button');
  btn.textContent = text;
  btn.addEventListener('click', onClick);
  return btn;
}

// الاستيراد: بدون {} وأي اسم تريده
import Button       from './Button.js';
import MyButton     from './Button.js';  // نفس الشيء

// الجمع بين الاثنين
import Button, { buttonStyles } from './Button.js';
NamedDefault
عدد التصديراتأي عددواحد فقط لكل ملف
الاستيراد{ name }أي اسم بدون {}
متى تستخدم؟دوال/ثوابت متعددةالشيء الرئيسي في الملف

.4 Dynamic Import Lazy Loading

تحميل module عند الحاجة فقط — يحسّن الأداء بشكل كبير:

⚡ import() الديناميكي
// تحميل module عند الضغط على زر
btn.addEventListener('click', async () => {
  // يُحمَّل فقط لما يضغط المستخدم
  const { heavyFunction } = await import('./heavy.js');
  heavyFunction();
});

// تحميل مشروط
const module = await import(
  isDark ? './dark-theme.js' : './light-theme.js'
);
فائدة: الصفحة تُحمَّل أسرع لأن الكود الثقيل لا يُحمَّل إلا عند الحاجة الفعلية.

.5 تنظيم مشروع حقيقي Project Structure

هيكل مشروع منظم يجعل التطوير أسرع والصيانة أسهل:

src/
  ├── components/ ← مكوّنات UI
  │   ├── Button.js
  │   ├── Modal.js
  │   └── Card.js
  ├── utils/ ← دوال مساعدة
  │   ├── api.js
  │   ├── storage.js
  │   └── helpers.js
  ├── data/ ← البيانات الثابتة
  │   └── constants.js
  └── main.js ← نقطة الدخول
📁 index.js — barrel export
// components/index.js — تجميع التصديرات
export { default as Button } from './Button.js';
export { default as Modal  } from './Modal.js';
export { default as Card   } from './Card.js';

// في main.js — استيراد نظيف
import { Button, Modal, Card } from './components/index.js';

.6 Single Responsibility One Job per Module

كل ملف (module) يجب أن يكون مسؤولاً عن شيء واحد فقط. هذا المبدأ يجعل الكود سهل الفهم والاختبار.

✅ تقسيم صحيح
// api.js — فقط التواصل مع الـ API
export async function fetchUsers() { ... }
export async function createUser(data) { ... }

// storage.js — فقط localStorage
export const storage = { get() {}, set() {}, remove() {} };

// ui.js — فقط تحديث الواجهة
export function renderUsers(users) { ... }
export function showError(msg) { ... }

// main.js — يجمّع كل شيء
import { fetchUsers }  from './api.js';
import { storage }     from './storage.js';
import { renderUsers } from './ui.js';

.7 الأنماط الشائعة Common Patterns

🔧 constants.js — الثوابت المركزية
// constants.js
export const API_URL   = 'https://api.myapp.com';
export const MAX_ITEMS = 50;
export const THEMES    = { LIGHT: 'light', DARK: 'dark' };
🎣 Custom Hook Pattern
// useCounter.js — منطق مستقل قابل لإعادة الاستخدام
export function createCounter(initial = 0) {
  let count = initial;
  return {
    get:       () => count,
    increment: () => ++count,
    decrement: () => --count,
    reset:     () => { count = initial; }
  };
}

// في main.js
import { createCounter } from './useCounter.js';
const counter = createCounter(10);
counter.increment();  // 11

📝 التمارين العملية Exercises

تمرين 1: تصدير Named

اكتب ملف greetings.js يصدّر دالتين: hello(name) تعيد "مرحباً name" وbye(name) تعيد "وداعاً name".

الحل: 
// greetings.js
export function hello(name) { return `مرحباً ${name}`; }
export function bye(name)   { return `وداعاً ${name}`; }

// main.js
import { hello, bye } from './greetings.js';
تمرين 2: Default Export

اكتب ملف Calculator.js يصدّر كائناً افتراضياً يحتوي على: add، subtract، multiply.

الحل: 
// Calculator.js
const Calculator = {
  add:      (a, b) => a + b,
  subtract: (a, b) => a - b,
  multiply: (a, b) => a * b,
};
export default Calculator;

// main.js
import Calculator from './Calculator.js';
تمرين 3: Barrel Export

عندك ملفات api.js و storage.js و helpers.js. اكتب index.js يصدّر كل شيء منهم.

الحل: 
// utils/index.js
export * from './api.js';
export * from './storage.js';
export * from './helpers.js';
تمرين 4: Import بأسماء مختلفة

استورد fetchData باسم getData و postData باسم sendData من ملف api.js.

الحل: 
import {
  fetchData as getData,
  postData  as sendData
} from './api.js';
تمرين 5: Dynamic Import

اكتب كوداً يحمّل module ثقيل chart.js فقط عند الضغط على زر "عرض الرسم البياني".

الحل: 
document.querySelector('#showChart')
  .addEventListener('click', async () => {
    const { renderChart } = await import('./chart.js');
    renderChart();
  });

اختبر فهمك للـ Modules!

1يمكن استخدام أكثر من export default في نفس الملف.
2Named imports تحتاج أقواس {} عند الاستيراد.
3Dynamic import يحسّن الأداء بتحميل الكود عند الحاجة فقط.
4Modules تعمل بدون مشاكل عند فتح الملف مباشرة بـ file://.
5يمكن استيراد default export بأي اسم تريده.
