.1 Sync vs Async Synchronous vs Asynchronous

JavaScript أحادية الخيط (single-threaded) — تنفّذ كوداً واحداً في وقت واحد. لكن بعض العمليات تأخذ وقتاً (طلبات شبكة، قراءة ملفات). هنا تأتي البرمجة غير المتزامنة.

☕ تخيل مطعم:

Synchronous: تطلب القهوة، تنتظر حتى تُحضَّر، ثم تطلب الطعام — كل شيء بالترتيب.
Asynchronous: تطلب القهوة والطعام في نفس الوقت، وتعمل على شيء آخر بينما يُحضَّران.

.2 Callbacks The Old Way

أقدم طريقة للتعامل مع الـ Async — تمرر دالة تُستدعى عند انتهاء العملية:

📞 Callback
// setTimeout — مثال بسيط على Async
console.log('1: قبل');

setTimeout(() => {
  console.log('2: بعد ثانيتين');
}, 2000);

console.log('3: بعد');

// النتيجة: 1 → 3 → 2 (مش بالترتيب!)
Callback Hell — مشكلة الـ Callbacks: 
// كل عملية تحتاج نتيجة السابقة → جحيم!
getUser(id, (user) => {
  getPosts(user.id, (posts) => {
    getComments(posts[0].id, (comments) => {
      // ومش خلصنا بعد... 😱
    });
  });
});

.3 Promises Promises

Promise هو كائن يمثّل نتيجة عملية مستقبلية — إما ستنجح (resolve) أو ستفشل (reject):

🤝 إنشاء Promise
const promise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve('تمت العملية!');   // نجاح
  } else {
    reject('حدث خطأ!');      // فشل
  }
});

// الاستخدام بـ .then و .catch
promise
  .then(result => console.log(result))   // 'تمت العملية!'
  .catch(error  => console.log(error))   // لو فشل
  .finally(() => console.log('انتهى'));  // دائماً ينفّذ
الحالةالمعنىالدالة
Pendingفي الانتظار—
Fulfilledنجحت.then()
Rejectedفشلت.catch()

.4 async / await Modern Async Syntax

الطريقة الحديثة — تجعل الكود الـ async يبدو sync وأسهل قراءة:

✨ async/await
// دالة async دائماً تعيد Promise
async function fetchData() {
  // await ينتظر Promise ينتهي قبل يكمل
  const user  = await getUser(1);
  const posts = await getPosts(user.id);
  return posts;
}

// Arrow async
const getData = async () => {
  const data = await somePromise();
  return data;
};
مقارنة: .then() vs async/await
🔄 نفس الوظيفة — طريقتان
// بـ .then()
fetch('https://api.example.com/users')
  .then(res  => res.json())
  .then(data => console.log(data));

// بـ async/await ✅ أوضح
async function getUsers() {
  const res  = await fetch('https://api.example.com/users');
  const data = await res.json();
  console.log(data);
}

.5 معالجة الأخطاء Error Handling

🛡️ try/catch مع async/await
async function fetchUser(id) {
  try {
    const res  = await fetch(`/api/users/${id}`);

    // تحقق من نجاح الاستجابة
    if (!res.ok) throw new Error(`HTTP Error: ${res.status}`);

    const data = await res.json();
    return data;

  } catch (error) {
    console.error('فشل الطلب:', error.message);
    return null;   // قيمة افتراضية عند الفشل

  } finally {
    console.log('انتهى الطلب — سواء نجح أم لا');
  }
}
res.ok هو true لو الـ status code بين 200-299. دائماً تحقق منه — fetch لا يرمي error تلقائياً عند 404 أو 500.

.6 Fetch API Making HTTP Requests

الأداة الأساسية لطلب البيانات من الإنترنت (APIs):

🌐 GET Request
async function getPosts() {
  try {
    const res   = await fetch('https://jsonplaceholder.typicode.com/posts');
    const posts = await res.json();
    console.log(posts.length, 'مقال');
  } catch (e) {
    console.error(e);
  }
}
📤 POST Request — إرسال بيانات
async function createPost(data) {
  const res = await fetch('https://jsonplaceholder.typicode.com/posts', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body:    JSON.stringify(data)
  });
  const result = await res.json();
  console.log('أُنشئ:', result);
}

createPost({ title: 'مقال جديد', body: 'المحتوى...' });

.7 Promise.all و Promise.race Multiple Promises

🏁 Promise.all — انتظر الكل
// تشغيل طلبات متعددة بالتوازي
async function loadAll() {
  const [users, posts, todos] = await Promise.all([
    fetch('/api/users').then(r => r.json()),
    fetch('/api/posts').then(r => r.json()),
    fetch('/api/todos').then(r => r.json()),
  ]);
  // ينتظر الثلاثة معاً — أسرع من الواحد تلو الآخر
}
الدالةالسلوكمتى تستخدم؟
Promise.all()ينتظر الكل — يفشل لو أي واحد فشلطلبات مستقلة لازمة كلها
Promise.allSettled()ينتظر الكل — لا يهتم بالفشلتريد نتيجة كل طلب حتى لو فشل
Promise.race()يُكمل بأول واحد ينتهيtimeout أو أسرع مصدر
Promise.any()يُكمل بأول نجاحجرب عدة مصادر

📝 التمارين العملية Exercises

تمرين 1: setTimeout كـ Promise

اكتب دالة delay(ms) تعيد Promise تنتهي بعد عدد معين من الميلي ثانية.

الحل: 
function delay(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

// استخدام
async function main() {
  console.log('بدأ');
  await delay(1000);
  console.log('بعد ثانية');
}
تمرين 2: جلب مستخدم

اكتب دالة async getUser(id) تجلب من https://jsonplaceholder.typicode.com/users/{id} وتعيد اسم المستخدم.

الحل: 
async function getUser(id) {
  const res  = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);
  const user = await res.json();
  return user.name;
}

getUser(1).then(name => console.log(name));
تمرين 3: معالجة الأخطاء

اكتب دالة async تجلب من URL وهمي وتعالج الخطأ بـ try/catch وتطبع رسالة مناسبة.

الحل: 
async function safeFetch(url) {
  try {
    const res = await fetch(url);
    if (!res.ok) throw new Error(`خطأ ${res.status}`);
    return await res.json();
  } catch (e) {
    console.error('فشل الطلب:', e.message);
    return null;
  }
}
تمرين 4: Promise.all

اجلب مستخدمين بـ id 1 و2 و3 في نفس الوقت باستخدام Promise.all وأعد مصفوفة أسمائهم.

الحل: 
async function getThreeUsers() {
  const base = 'https://jsonplaceholder.typicode.com/users';
  const users = await Promise.all([
    fetch(`${base}/1`).then(r => r.json()),
    fetch(`${base}/2`).then(r => r.json()),
    fetch(`${base}/3`).then(r => r.json()),
  ]);
  return users.map(u => u.name);
}
تمرين 5: عرض loading

اكتب دالة تُظهر "جاري التحميل..." قبل الطلب وتُخفيها بعده (سواء نجح أم فشل).

الحل: 
async function fetchWithLoader(url) {
  const loader = document.getElementById('loader');
  loader.style.display = 'block';
  try {
    const res  = await fetch(url);
    const data = await res.json();
    return data;
  } finally {
    loader.style.display = 'none';
  }
}
