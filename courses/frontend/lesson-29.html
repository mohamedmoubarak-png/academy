.1 ما هو Deployment؟ What is Deployment?

Deployment = نشر موقعك على الإنترنت ليصبح متاحاً للجميع بـ URL حقيقي.

🏪 تخيل محل تجاري:

كودك على جهازك = بضاعة في المخزن. الـ Deployment = فتح المحل للعملاء. السيرفر = المحل نفسه الذي يعمل 24/7. الـ URL = عنوان المحل.

أنواع المواقع والاستضافة:
النوعالمثالالاستضافة المناسبة
Static Siteموقع شخصي، PortfolioNetlify, GitHub Pages, Vercel
SPA (React/Vue)تطبيق ويب تفاعليVercel, Netlify
Full-Stackموقع مع Backend/DBRailway, Render, DigitalOcean
WordPressموقع بـ CMSHostinger, SiteGround

.2 Netlify Easiest Deploy

🟢
Netlify
أسهل وأسرع طريقة لنشر المواقع الثابتة. اسحب وأفلت مجلدك وموقعك يعمل خلال ثوانٍ!
مجاني للمشاريع الشخصية Static Sites
طريقة النشر الأولى: Drag & Drop
سجّل على netlify.com
اضغط "Sign up" واستخدم حساب GitHub
اسحب مجلدك للصفحة الرئيسية
اسحب مجلد المشروع كاملاً إلى منطقة "Drag and drop"
موقعك جاهز! 🎉
ستحصل على رابط مثل: random-name.netlify.app
طريقة النشر الثانية: ربط GitHub (الأفضل)
⚡ Netlify CLI
# تثبيت Netlify CLI
npm install -g netlify-cli

# تسجيل الدخول
netlify login

# نشر من المجلد الحالي
netlify deploy --prod

# أو ربط المشروع بـ GitHub للنشر التلقائي
netlify init

.3 Vercel Modern Deploy

▲
Vercel
الخيار الأول لمشاريع Next.js وReact. أداء استثنائي مع Edge Network عالمي.
مجاني للأفراد React / Next.js / Vue
ارفع مشروعك على GitHub
تأكد من وجود كودك في repo على GitHub
سجّل على vercel.com بحساب GitHub
اضغط "Continue with GitHub"
Import المشروع
اختر الـ repo من قائمة مشاريعك — Vercel يكتشف الإطار تلقائياً
Deploy! 🚀
موقعك على project.vercel.app خلال دقيقة واحدة
⚡ Vercel CLI
# تثبيت
npm install -g vercel

# نشر (يسألك أسئلة بسيطة)
vercel

# نشر مباشر للـ production
vercel --prod

.4 GitHub Pages Free with GitHub

🐙
GitHub Pages
استضافة مجانية مدمجة مع GitHub. مثالية للـ Portfolio والمشاريع الثابتة.
مجاني تماماً Static HTML/CSS/JS
ارفع مشروعك على GitHub
Settings → Pages في الـ Repo
اختر Source: Deploy from branch → main
موقعك على: username.github.io/repo-name
📦 نشر مشروع Vite على GitHub Pages
# 1. أضف للـ vite.config.js
export default {
  base: '/repo-name/'
}

# 2. بناء المشروع
npm run build

# 3. نشر مجلد dist
npm install -D gh-pages

# 4. أضف script في package.json
"deploy": "gh-pages -d dist"

# 5. نشر
npm run deploy

.5 Domain مخصص Custom Domain

بدل mysite.netlify.app، تقدر تستخدم www.mysite.com بخطوات بسيطة:

اشتر اسم النطاق
من: Namecheap, GoDaddy, Cloudflare — اختر اسم مناسب (.com, .io, .dev)
أضف Domain في لوحة Netlify/Vercel
Domain Management → Add Domain → أدخل اسمك
عدّل DNS Records عند مزود النطاق
أضف CNAME أو A Record حسب تعليمات المنصة
انتظر انتشار الـ DNS (24-48 ساعة)
تحقق بـ: whatsmydns.net
HTTPS مجاناً! Netlify وVercel يوفّران شهادة SSL مجانية تلقائياً (Let's Encrypt). موقعك سيكون https:// بدون أي إعداد إضافي.

.6 Environment Variables Env Vars

المتغيرات الحساسة (مفاتيح API، كلمات مرور) لا تُكتب في الكود — تُخزَّن كـ Environment Variables.

🔒 ملف .env
# ملف .env (لا ترفعه لـ GitHub!)
VITE_API_KEY=abc123secret
VITE_API_URL=https://api.myservice.com
DATABASE_URL=postgresql://user:pass@host/db
📖 قراءة المتغيرات في Vite
// في كودك (Vite يتعامل مع VITE_ prefix)
const apiKey = import.meta.env.VITE_API_KEY;
const apiUrl = import.meta.env.VITE_API_URL;
إضافة Env Vars في Netlify/Vercel:
  • Netlify: Site settings → Build & deploy → Environment variables
  • Vercel: Project Settings → Environment Variables
  • أضف نفس المتغيرات الموجودة في .env المحلي
أهم قاعدة: أضف .env و.env.local لملف .gitignore دائماً. لا ترفع أي مفاتيح API لـ GitHub!

.7 CI/CD: النشر التلقائي Continuous Deployment

CI/CD = نشر تلقائي عند كل push لـ GitHub. أقوى ميزة في الـ Deployment الحديث!

🤖 كيف يعمل:

أنت تكتب كوداً → git push → Netlify/Vercel يكتشف التغيير تلقائياً → يبني الموقع → ينشره خلال دقيقتين. بدون أي تدخل منك!

🔄 GitHub Actions (CI/CD يدوي)
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: 20
      - run: npm install
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

🚀 موقع Portfolio بسيط جاهز للنشر

📝 التمارين العملية Exercises

تمرين 1: اختيار المنصة

لديك ثلاثة مشاريع: (أ) موقع Portfolio بـ HTML/CSS، (ب) تطبيق React، (ج) تطبيق Node.js مع قاعدة بيانات. أي منصة تختار لكل منها؟

الحل:

(أ) Portfolio HTML/CSS → GitHub Pages (مجاني وبسيط)
(ب) React App → Vercel أو Netlify (ممتازان للـ SPA)
(ج) Node.js + DB → Railway أو Render (يدعمان الـ Backend)

تمرين 2: ملف .gitignore للنشر

اكتب محتوى .gitignore لمشروع Vite يتجاهل: node_modules، ملف .env، ومجلد dist.

الحل: 
node_modules/
.env
.env.local
dist/
تمرين 3: Environment Variables

لديك مفتاح API اسمه WEATHER_KEY. اكتب كيف تحفظه في .env وكيف تقرأه في مشروع Vite.

الحل: 
# في ملف .env
VITE_WEATHER_KEY=your_api_key_here

// في الكود JavaScript
const apiKey = import.meta.env.VITE_WEATHER_KEY;
تمرين 4: بناء مشروع Vite للنشر

اكتب الأوامر لبناء مشروع Vite وتحضيره للنشر على Netlify عبر Drag & Drop.

الحل: 
# بناء المشروع (ينشئ مجلد dist/)
npm run build

# بعد كذا، اسحب مجلد dist/ لصفحة Netlify
# أو استخدم CLI:
netlify deploy --prod --dir=dist
تمرين 5: CI/CD مع GitHub

ما هي الخطوات لربط مشروع React بـ Vercel للنشر التلقائي عند كل push؟

الحل:

1. ارفع المشروع على GitHub repo
2. سجّل في vercel.com بحساب GitHub
3. اضغط "Add New Project" واختر الـ repo
4. Vercel يكتشف React ويعدّ Build Settings تلقائياً
5. اضغط Deploy — كل push لـ main ينشر تلقائياً!

اختبر فهمك للـ Deployment!

1يجب رفع ملف .env لـ GitHub ليعمل الموقع على السيرفر.
2Netlify وVercel يوفّران شهادة HTTPS مجانية تلقائياً.
3CI/CD يعني أن الموقع ينشر تلقائياً عند كل push لـ GitHub.
4GitHub Pages مناسب لنشر تطبيقات Backend مع قواعد بيانات.
5متغيرات البيئة في Vite يجب أن تبدأ بـ VITE_ لتكون متاحة في الكود.
