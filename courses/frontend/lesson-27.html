.1 ما هو NPM؟ What is NPM?

NPM = Node Package Manager. هو مدير الحزم الرسمي لـ JavaScript. يتيح لك:

  • تثبيت مكتبات جاهزة بأمر واحد (مثل React, Axios, Lodash)
  • مشاركة كودك كمكتبة مع الآخرين
  • تشغيل scripts لأتمتة المهام (Build, Test, Deploy)
  • إدارة إصدارات المكتبات في مشروعك
🛒 تخيل سوبر ماركت للكود:

npmjs.com = السوبر ماركت. الحزمة (Package) = منتج جاهز. npm install = إضافة للسلة وتحميل. بدل ما تكتب كل شيء من الصفر، تشتري المكتبات الجاهزة!

npm بالأرقام:
+2 مليون حزمة متاحة على npmjs.com
+20 مليار تحميل شهرياً

.2 التثبيت والتحقق Installation

تثبيت Node.js (يأتي مع NPM تلقائياً):
# حمّل من: nodejs.org (اختر LTS)
# بعد التثبيت، تحقق:
node --version
v20.11.0
npm --version
10.2.4
بدء مشروع جديد:
🚀 npm init
# طريقة تفاعلية (يسألك أسئلة)
npm init

# طريقة سريعة (يقبل الافتراضيات كلها)

npm init -y هي الطريقة الأسرع. تنشئ package.json فوراً بدون أسئلة — يمكنك التعديل لاحقاً.

.3 ملف package.json The Manifest

package.json هو "بطاقة هوية" مشروعك. يحتوي على كل معلوماته ومكتباته:

{
  "name": "my-website",
  "version": "1.0.0",
  "description": "موقعي الشخصي",
  "scripts": {
    "start": "node index.js",
    "dev": "vite",
    "build": "vite build"
  },
  "dependencies": {
    "axios": "^1.6.0",
    "lodash": "^4.17.21"
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}
الإصدارات (Semantic Versioning):
الصيغةالمعنىمثال
1.2.3إصدار محدد بالضبطMajor.Minor.Patch
^1.2.3أي إصدار 1.x.x (الأشهر)1.3.0 ✅، 2.0.0 ❌
~1.2.3أي إصدار 1.2.x فقط1.2.5 ✅، 1.3.0 ❌
*أي إصدار (خطر!)غير مستحسن

.5 Scripts: أتمتة المهام NPM Scripts

في package.json، قسم "scripts" يتيح لك تشغيل أوامر بأسماء قصيرة:

⚙️ تعريف Scripts
// في package.json
{
  "scripts": {
    "start":  "node server.js",
    "dev":    "vite --open",
    "build":  "vite build",
    "test":   "jest",
    "lint":   "eslint src/",
    "format": "prettier --write ."
  }
}
▶️ تشغيل Scripts
# start و test لهما اختصار:
npm start
npm test

# غيرها تحتاج "run":
npm run dev
npm run build
npm run lint
npm run format
لماذا Scripts؟ بدل ما تحفظ الأمر الطويل كل مرة، تكتبه مرة في package.json وتشغّله باختصار. ويعمل بنفس الطريقة على أي جهاز في الفريق!

📦 إدارة الحزم
# رفع إصدار حزمة
npm update axios

# حذف حزمة

# رؤية الحزم المثبتة
npm list

ملف package-lock.json:
لا تحذفه! package-lock.json يضمن أن كل أعضاء الفريق يثبّتون نفس الإصدارات بالضبط. ارفعه دائماً مع مشروعك.

.7 npx وبدائل NPM npx & Alternatives

npx — تشغيل بدون تثبيت:
⚡ npx

📝 التمارين العملية Exercises

تمرين 1: بدء مشروع

اكتب الأوامر لإنشاء مجلد my-app، الدخول إليه، ثم تهيئة مشروع npm بدون أسئلة.

الحل: 
mkdir my-app
cd my-app
npm init -y
تمرين 2: تثبيت حزم

ثبّت axios كـ dependency عادية، و vite كـ devDependency.

الحل: 
npm install axios
npm install -D vite
تمرين 3: إضافة Scripts

أضف لـ package.json script باسم "dev" يشغّل vite، وscript باسم "build" يشغّل vite build.

الحل: 
{
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  }
}

ثم تشغّل: npm run dev أو npm run build

تمرين 4: حذف وتحديث

احذف حزمة lodash، ثم حدّث حزمة axios لأحدث إصدار.

الحل: 
npm uninstall lodash
npm update axios
تمرين 5: npx واستنساخ مشروع

اكتب الأمر لإنشاء مشروع Vite باسم portfolio باستخدام npx. ثم اكتب الأوامر لتثبيت الحزم بعد استنساخ مشروع من GitHub.

الحل: 
# إنشاء مشروع Vite
npx create vite@latest portfolio

# بعد clone: تثبيت الحزم
git clone https://github.com/user/project.git
cd project
npm install

4يجب رفع مجلد node_modules لـ GitHub مع المشروع.
