.1 لماذا نحتاج Express؟ Why Express?

في الدرس السابق بنينا خادماً كاملاً بـ HTTP خام — لكن لاحظنا المشكلة: كود مطوّل ومتكرر في كل مشروع. Express يحل هذه المشكلة.

🚂
Express = غطاء ذكي على Node.js HTTP

Express لا يُعيد اختراع العجلة — هو يُكمل ما بنيناه. تحت غطائه يعمل نفس http.createServer() الذي تعلمناه، لكن مع routing أنيق ومئات الـ helper functions الجاهزة.

المهمةHTTP خام (الدرس السابق)Express (هذا الدرس)
Route بسيطif(method==='GET'&&url==='/')app.get('/', fn)
قراءة Bodyدالة readBody — 15 سطرreq.body مباشرة
إعادة JSONwriteHead + JSON.stringify + endres.json(data)
كود 404res.writeHead(404) + res.end()res.status(404).json()
Params ديناميكيةurl.split('/') يدوياًreq.params.id
حجم الكود~50 سطر لخادم بسيط~20 سطر لنفس الخادم
Express = أكثر إطار Node.js شيوعاً في العالم. يُستخدم في Netflix, Uber, IBM, وآلاف الشركات. تعلّمه يفتح أمامك سوقاً ضخماً من فرص العمل.

.3 أول خادم Express First Express Server

مقارنة مباشرة بين HTTP خام وExpress — نفس النتيجة، كود أقصر وأوضح:

🚂 app.js — أبسط خادم Express
const express = require('express');
const app = express();  // ← إنشاء التطبيق

// Route: GET /
app.get('/', (req, res) => {
  res.json({ message: 'مرحباً من Express! 🚂' });
});

// Route: GET /api/users
app.get('/api/users', (req, res) => {
  const users = [{ id: 1, name: 'أحمد' }, { id: 2, name: 'سارة' }];
  res.json({ data: users, count: users.length });
});

// ابدأ الاستماع على port 3000
app.listen(3000, () => {
  console.log('✅ Express يعمل على http://localhost:3000');
});
▶️ تشغيل الخادم
node app.js
# → ✅ Express يعمل على http://localhost:3000

# اختبر:
curl http://localhost:3000
# → {"message":"مرحباً من Express! 🚂"}

curl http://localhost:3000/api/users
# → {"data":[...],"count":2}
الفرق واضح: لا writeHead، لا JSON.stringify، لا res.end() يدوي. Express يتكفل بكل ذلك عبر res.json().

.4 app.get / app.post / app.use Routing Methods

لكل HTTP Method دالة مقابلة في Express:

app.get()
جلب البيانات — READ
app.post()
إنشاء مورد جديد — CREATE
app.put()
تحديث كامل — REPLACE
app.patch()
تحديث جزئي — UPDATE
app.delete()
حذف المورد — DELETE
app.use()
جميع الطرق — Middleware
🗺️ Routes كاملة لـ API مستخدمين
// الـ middleware يأتي أولاً (قبل الـ routes)
app.use(express.json());

// ─── CRUD كامل ───
app.get('/api/users',        getAll);    // جلب الكل
app.get('/api/users/:id',    getOne);    // جلب واحد
app.post('/api/users',       create);    // إنشاء
app.put('/api/users/:id',    replace);   // تحديث كامل
app.patch('/api/users/:id',  update);    // تحديث جزئي
app.delete('/api/users/:id', remove);    // حذف

// كل route تأخذ: (path, ...middleware, handler)
function getAll(req, res) {
  res.json({ users: [] });
}
⚙️ الفرق بين app.get() وapp.use()
// app.get() ← طلبات GET فقط، مسار محدد
app.get('/api/users', (req, res) => { ... });

// app.use() ← جميع الطرق، مسار وكل ما بعده
app.use('/api', someMiddleware);   // يُطبَّق على /api/*
app.use(express.json());           // بدون مسار = كل الطلبات

// مثال: 404 handler يُوضع آخر شيء
app.use((req, res) => {
  res.status(404).json({ error: 'Not Found', path: req.url });
});

.5 req.params, req.query, req.body Request Data

بيانات الطلب تأتي من ثلاثة أماكن — افهم الفرق جيداً:

req.params
/api/users/:id/posts/:postId

قيم في المسار نفسه — تُعرَّف بـ :

req.params.id = "42"
req.params.postId = "7"
req.query
/api/users?page=2&sort=name

ما بعد ? في الـ URL — اختياري

req.query.page = "2"
req.query.sort = "name"
req.body
POST /api/users
Body: {"name":"أحمد"}

بيانات POST/PUT — يحتاج express.json()

req.body.name = "أحمد"
req.body.email = "..."
🔍 أمثلة عملية على الثلاثة
// 1. req.params — الـ :id في المسار
app.get('/api/users/:id', (req, res) => {
  const { id } = req.params;      // /api/users/42 → id = "42"
  res.json({ userId: id });
});

// 2. req.query — ما بعد ؟
app.get('/api/users', (req, res) => {
  const { page = 1, limit = 10, sort = 'id' } = req.query;
  // /api/users?page=2&limit=5&sort=name
  res.json({ page, limit, sort });
});

// 3. req.body — بيانات POST
app.post('/api/users', (req, res) => {
  const { name, email, age } = req.body;
  // Body: {"name":"أحمد","email":"ahmed@mail.com","age":25}
  if (!name || !email) {
    return res.status(400).json({ error: 'الاسم والبريد مطلوبان' });
  }
  res.status(201).json({ message: 'تم الإنشاء', data: { name, email } });
});
ملاحظة مهمة: قيم req.params وreq.query دائماً strings حتى لو بدت أرقاماً. استخدم Number(req.params.id) أو parseInt(req.params.id) لتحويلها.

.6 express.json() — قراءة الـ Body Body Parsing

express.json() هو middleware مدمج يقرأ الـ body JSON ويضعه في req.body تلقائياً — بدله req.body سيكون undefined:

RequestPOST /api/users
→
express.json()يقرأ الـ body
→
Route Handlerreq.body جاهز
→
Responseres.json()
⚙️ موضع express.json() مهم جداً
const express = require('express');
const app = express();

// ✅ قبل الـ routes — هذا هو الصحيح
app.use(express.json());

// الآن req.body متاح في جميع الـ routes
app.post('/api/users', (req, res) => {
  console.log(req.body);  // { name: 'أحمد', email: '...' }
  res.status(201).json({ received: req.body });
});

// ❌ بدون express.json() أو بعد الـ routes:
// req.body سيكون undefined دائماً
📋 Middlewares مدمجة مفيدة
// لقراءة JSON body
app.use(express.json());

// لقراءة Form body (application/x-www-form-urlencoded)
app.use(express.urlencoded({ extended: true }));

// لتقديم ملفات ثابتة (HTML, CSS, صور)
app.use(express.static('public'));

// يمكن تحديد مسار للـ static files
app.use('/files', express.static('uploads'));
Content-Type مهم: express.json() يعمل فقط عندما يُرسل العميل Content-Type: application/json. إذا لم يُرسله، سيظل req.body فارغاً حتى مع وجود الـ middleware.

.7 res.json(), res.status(), res.send() Response Methods

Express يُبسّط الرد على الطلبات بدوال واضحة:

res.json(data) إرسال JSON — يعيّن Content-Type تلقائياً. الأفضل للـ APIs.
res.send(data) إرسال أي نوع — string, Buffer, object. يكتشف النوع تلقائياً.
res.status(code) يعيّن status code — يُسلسَل مع .json() أو .send().
res.sendStatus(code) يرسل الكود فقط بدون body — مثالي لـ 204 No Content.
res.redirect(url) تحويل المستخدم لرابط آخر بكود 302 (أو 301 للدائم).
💡 أمثلة عملية على جميع الطرق
// ✅ النجاح — أفضل طريقة للـ APIs
res.json({ data: user, success: true });

// ✅ مع status code — الأكثر استخداماً
res.status(201).json({ message: 'تم الإنشاء', data: newUser });
res.status(200).json({ data: users });

// ✅ الأخطاء
res.status(400).json({ error: 'بيانات خاطئة' });
res.status(401).json({ error: 'يجب تسجيل الدخول' });
res.status(403).json({ error: 'ليس لديك صلاحية' });
res.status(404).json({ error: 'غير موجود' });
res.status(500).json({ error: 'خطأ داخلي' });

// ✅ بدون body (مثلاً بعد الحذف)
res.sendStatus(204);

// ✅ تحويل
res.redirect('/api/v2/users');     // 302
res.redirect(301, '/new-path');    // 301 دائم
⚠️ لا تستدعِ دالتين للرد في نفس الـ route:
res.json() ثم res.send() = خطأ "Cannot set headers after they are sent". استخدم return قبل أي رد داخل شرط.

.8 تنظيم المشروع + Express في العمل Project Structure & Live

المشاريع الحقيقية تُقسّم الـ routes في ملفات منفصلة باستخدام express.Router():

🚂 Express Server — منظّم بـ Router app.js + routes/ 
// ── routes/users.js ──────────────────
const router = require('express').Router();

const users = [
  { id:1, name:'أحمد' },
  { id:2, name:'سارة' },
];

router.get('/', (req, res) => {
  res.json({ data: users });
});

router.get('/:id', (req, res) => {
  const u = users.find(
    x => x.id === +req.params.id
  );
  if(!u) return res
    .status(404)
    .json({error:'غير موجود'});
  res.json({ data: u });
});

router.post('/', (req, res) => {
  const { name, email } = req.body;
  if(!name)
    return res.status(400)
      .json({error:'الاسم مطلوب'});
  const u = {
    id: Date.now(), name, email
  };
  users.push(u);
  res.status(201).json({data:u});
});

module.exports = router;

// ── app.js ───────────────────────────
const express = require('express');
const app = express();
const usersRouter =
  require('./routes/users');

app.use(express.json());
app.use('/api/users', usersRouter);

app.use((req, res) =>
  res.status(404).json(
    {error:'Not Found'}
  )
);

app.listen(3000, () =>
  console.log('🚂 يعمل :3000')
);
node app.js

  ├── app.js              ← نقطة الدخول + middleware
  ├── routes/
  │   ├── users.js       ← GET/POST/PUT /api/users

express.Router(): يُمكّنك من تجميع routes مرتبطة في ملف واحد. كل router له use, get, post... خاصة به. في app.js تربطه بمسار معيّن بـ app.use('/api/users', usersRouter) — وتلقائياً كل routes داخله تبدأ بهذا المسار.

📝 التمارين العملية Exercises

تمرين 1: أول Express App

أنشئ Express app بثلاثة routes:
• GET / → JSON: { page: "home", powered: "Express" }
• GET /about → JSON: { page: "about", version: "1.0" }
• 404 handler → JSON: { error: "Not Found", path: req.url }

الحل: 
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.json({ page: 'home', powered: 'Express' });
});

app.get('/about', (req, res) => {
  res.json({ page: 'about', version: '1.0' });
});

// 404 handler — يجب أن يكون آخر شيء
app.use((req, res) => {
  res.status(404).json({ error: 'Not Found', path: req.url });
});

app.listen(3000, () => console.log('✅ يعمل'));
تمرين 2: Route بـ params

أنشئ GET /api/products/:id يُعيد بيانات منتج من مصفوفة وهمية. إذا لم يوجد المنتج أعد 404.

الحل: 
const products = [
  { id: 1, name: 'لابتوب', price: 3000 },
  { id: 2, name: 'جوال',   price: 1500 },
];

app.get('/api/products/:id', (req, res) => {
  const product = products.find(p => p.id === Number(req.params.id));
  if (!product) {
    return res.status(404).json({ error: 'المنتج غير موجود' });
  }
  res.json({ data: product });
});
تمرين 3: Route مع query params

أنشئ GET /api/search يقرأ query string:
?q=keyword&page=1&limit=10
ويرد بـ JSON يُظهر القيم المُستقبلة مع قيم افتراضية.

الحل: 
app.get('/api/search', (req, res) => {
  const {
    q = '',
    page = '1',
    limit = '10',
  } = req.query;

  res.json({
    query:   q,
    page:    Number(page),
    limit:   Number(limit),
    results: [],  // مكان نتائج البحث
  });
});
تمرين 4: POST route مع Body

أنشئ POST /api/messages يستقبل { text } ويُعيد:
{ id, text, createdAt } مع status 201.
إذا لم يوجد text: أعد 400.

الحل: 
app.use(express.json());  // قبل الـ routes

app.post('/api/messages', (req, res) => {
  const { text } = req.body;

  if (!text || text.trim() === '') {
    return res.status(400).json({ error: 'النص مطلوب' });
  }

  const message = {
    id:        Date.now(),
    text:      text.trim(),
    createdAt: new Date().toISOString(),
  };

  res.status(201).json({ data: message });
});
تمرين 5: تنظيم بـ Router

انقل routes تمارين 2 و3 و4 إلى ملف routes/api.js باستخدام express.Router()، ثم استورده في app.js بمسار /api.

الحل: 
// routes/api.js
const router = require('express').Router();

router.get('/products/:id', (req, res) => { ... });
router.get('/search',       (req, res) => { ... });
router.post('/messages',    (req, res) => { ... });

module.exports = router;

// app.js
const express = require('express');
const app = express();
const apiRouter = require('./routes/api');

app.use(express.json());
app.use('/api', apiRouter);  // كل routes تبدأ بـ /api

app.listen(3000, () => console.log('✅ يعمل'));

// النتيجة: /api/products/:id, /api/search, /api/messages
