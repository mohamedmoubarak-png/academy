.1 كيف يعمل الإنترنت؟ How Internet Works

عندما تفتح أي موقع مثل google.com — ماذا يحدث خلف الكواليس؟

🤔
تشبيه المطعم

الإنترنت مثل مطعم — أنت الزبون (Client) تطلب طبقاً من النادل الذي يذهب للـ مطبخ (Server) ويُعيد لك الطعام. أنت لا ترى المطبخ — هذا هو الباك إند.

الخطوات الفعلية عند كتابة google.com:

  • المتصفح يبحث عن عنوان IP لـ google.com عبر نظام DNS
  • يُرسل طلب HTTP لخادم Google
  • الخادم يُعالج الطلب ويُرجع استجابة HTML
  • المتصفح يُحوّل الـ HTML لصفحة مرئية

.2 Client vs Server Client-Server Model

الجانبClient (العميل)Server (الخادم)
من هو؟المتصفح / التطبيقحاسب يستقبل الطلبات
يعمل أين؟جهاز المستخدممركز بيانات / سحابة
يرى الكود؟نعم (HTML, CSS, JS)لا — مخفي تماماً
يبدأ التواصل؟دائماً هو يبدأينتظر ويستجيب
يصل للـ DB؟لا (مباشرة)نعم
أمثلةChrome, Safari, React Native AppNode.js, Python, Java
الخادم يمكنه أن يكون Client في نفس الوقت! مثلاً: خادمك يُرسل طلباً لخدمة الدفع (Stripe) — فهو عميل بالنسبة لـ Stripe وخادم بالنسبة لتطبيقك.

.3 HTTP — لغة الويب HTTP Protocol

HTTP (HyperText Transfer Protocol) هي اللغة التي يتحدث بها Client وServer — كل طلب واستجابة تتبع هذا البروتوكول.

🌐 هيكل طلب HTTP (Request)
// ما يُرسله المتصفح للخادم
GET /api/users/42 HTTP/1.1
Host: api.example.com
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
Content-Type: application/json
Accept: application/json

// ↑ السطر الأول: الطريقة + المسار + الإصدار
// ↑ Headers: معلومات إضافية
// Body (اختياري): في POST/PUT تُرسل البيانات هنا
📨 هيكل استجابة HTTP (Response)
// ما يُعيده الخادم
HTTP/1.1 200 OK
Content-Type: application/json
X-Request-Id: 8f3a2b91

{
  "id": 42,
  "name": "أحمد محمد",
  "email": "ahmed@example.com"
}

// ↑ السطر الأول: الإصدار + كود الحالة
// ↑ Headers: نوع المحتوى ومعلومات الخادم
// ↑ Body: البيانات المُعادة (عادةً JSON)

.4 أساليب HTTP وأكواد الاستجابة Methods & Status Codes

كل طلب HTTP له طريقة (Method) تُخبر الخادم بنوع العملية المطلوبة:

GET — اقرأ POST — أنشئ PUT — استبدل PATCH — عدّل جزئياً DELETE — احذف
🗂️ HTTP Methods في REST API
// مثال: API إدارة المستخدمين

GET    /api/users        → جلب قائمة المستخدمين
GET    /api/users/42     → جلب مستخدم محدد
POST   /api/users        → إنشاء مستخدم جديد
PUT    /api/users/42     → تحديث المستخدم كاملاً
PATCH  /api/users/42     → تحديث حقل واحد فقط
DELETE /api/users/42     → حذف المستخدم

وأكواد الاستجابة تُخبرك بنتيجة الطلب:

200 OK
نجاح — جلب/تحديث
201 Created
تم إنشاء مورد جديد
204 No Content
نجاح بدون بيانات (حذف)
301 Redirect
تحويل دائم لرابط آخر
400 Bad Request
خطأ في بيانات الطلب
401 Unauthorized
غير مُصادق عليه
403 Forbidden
مُصادق لكن غير مُخوّل
404 Not Found
المورد غير موجود
500 Server Error
خطأ داخلي في الخادم
قاعدة سهلة: 2xx = نجاح ✅ | 3xx = تحويل 🔀 | 4xx = خطأ من العميل ❌ | 5xx = خطأ من الخادم 💥

.5 الفرونت إند vs الباك إند Frontend vs Backend

الجانب🎨 الفرونت إند⚙️ الباك إند
التعريفما يراه ويتفاعل معه المستخدمالمنطق والبيانات خلف الكواليس
يعمل أين؟المتصفح / الجهازخادم في السحابة
اللغاتHTML, CSS, JS, React, FlutterNode.js, Python, Java, Go
يتعامل مع؟واجهة المستخدم، الأنيميشنقواعد البيانات، الأمان، المنطق
المخرجصفحات، تطبيقاتAPIs تُخدم أي Client
مثال يوميشكل وتفاعل موقع أمازونسلة التسوق، المدفوعات، المخزون
الباك إند الواحد يخدم أي عدد من الـ Clients — موقع ويب، تطبيق iOS، تطبيق Android، Alexa — كلها تتصل بنفس الـ API!

📝 التمارين العملية Exercises

تمرين 1: تمييز العناصر

صنّف كل عنصر من التالي: (Client / Server / Database)
أ) تطبيق Instagram على هاتفك
ب) MySQL المثبّت على خادم AWS
ج) خادم Node.js يعالج طلبات التسجيل

الحل:

أ) Client — التطبيق يعمل على جهازك ويُرسل الطلبات
ب) Database — يخزن ويسترجع البيانات
ج) Server — يستقبل الطلبات ويعالجها

تمرين 2: صمّم API

صمّم نقاط API لتطبيق مدونة يحتوي على منشورات (posts). اكتب الـ Method والمسار لكل عملية: جلب الكل، جلب واحد، إنشاء، تعديل، حذف.

الحل: 
GET    /api/posts          → جلب جميع المنشورات
GET    /api/posts/:id      → جلب منشور واحد
POST   /api/posts          → إنشاء منشور جديد
PUT    /api/posts/:id      → تحديث المنشور كاملاً
DELETE /api/posts/:id      → حذف المنشور
تمرين 3: أكواد الحالة

ما كود HTTP المناسب لكل حالة؟
أ) جلب ملف شخصي بنجاح
ب) محاولة حذف منشور لا تملكه (مُسجّل لكن غير مخوّل)
ج) إرسال بيانات تسجيل ناقصة
د) خطأ غير متوقع في الخادم

الحل:

أ) 200 OK — النجاح العادي
ب) 403 Forbidden — مُصادق لكن غير مُخوّل
ج) 400 Bad Request — خطأ في بيانات الطلب
د) 500 Internal Server Error — خطأ داخلي

تمرين 4: رحلة الطلب

رتّب الخطوات التالية بالترتيب الصحيح عندما يضغط مستخدم "اشتري الآن":
(أ) الخادم يُخصم من المخزون في قاعدة البيانات
(ب) التطبيق يُرسل POST /api/orders مع بيانات الطلب
(ج) المستخدم يرى تأكيد الطلب على الشاشة
(د) الخادم يُعيد 201 Created مع رقم الطلب
(هـ) الخادم يتحقق من صحة بيانات الدفع

الحل:

الترتيب الصحيح: ب ← هـ ← أ ← د ← ج
(إرسال الطلب ← التحقق من الدفع ← تحديث المخزون ← إعادة الاستجابة ← عرض التأكيد)

تمرين 5: فرونت أم باك؟

حدّد لكل مهمة: هل هي مسؤولية الفرونت إند أم الباك إند؟
أ) التحقق من صحة بريد المستخدم في قاعدة البيانات
ب) إظهار رسالة خطأ حمراء عند إدخال كلمة مرور خاطئة
ج) تشفير كلمة المرور قبل التخزين
د) تنسيق تاريخ إنشاء المنشور بالعربي
هـ) إرسال إيميل ترحيب للمستخدم الجديد

الحل:

أ) باك إند — الوصول لقاعدة البيانات
ب) فرونت إند — عرض للمستخدم
ج) باك إند — الأمان الحساس
د) فرونت إند — تنسيق مرئي
هـ) باك إند — إرسال البريد من الخادم
